## 1. システム概要

プレイヤーがパーティに配備する3体のモンスターの **種族（Tribe）** の組み合わせによって、
戦闘中にパーティ全体（主人公アルド + 3体）へ適用されるパッシブバフ群が変化する。

編成の自由度と「シナジーを狙う楽しさ」を与えつつ、
完全3種族統一より「2+1の折衷」にも競争力を持たせるために **3層の設計** を採用する。

```
Layer 1: 部分シナジー  → 同一種族 2体でトリガー（弱いが確保しやすい）
Layer 2: 完全シナジー  → 同一種族 3体でトリガー（最大効果）
Layer 3: クロス共鳴    → 特定の2種族の組み合わせでトリガー（部分シナジーと加算）
```

**設計意図**:
- 3体完全統一: Layer2 の最大値を享受するが、クロス共鳴が発動しない
- 2+1 編成: Layer1（1種族）+ Layer3（クロス共鳴）が発動 → 一方に特化した完全統一と競争できる

## 2. 種族の拡張定義

現行実装の4種族に **DRAGON（竜族）** と **ORC（鬼人族）** を追加し、合計6種族とする。

```typescript
// src/types/game.ts — 現行から変更
type Tribe = 'UNDEAD' | 'DEMON' | 'BEAST' | 'HUMANOID' | 'DRAGON' | 'ORC';
```

| 種族 | 和名 | キャラクター | 強み |
|---|---|---|---|
| UNDEAD | アンデッド | 死者の兵士・骸骨・ゾンビ | 耐久・HP回復・継続戦 |
| DEMON | 悪魔族 | 悪魔・インプ・堕天使 | 闇属性・デバフ・状態異常 |
| BEAST | 獣族 | 狼・熊・虎・ゴブリン | 高速・物理火力・先制 |
| HUMANOID | 人族 | 騎士・盗賊・弓使い | 会心・オールラウンド |
| DRAGON | 竜族 | 竜・ワイバーン・竜神 | 属性火力・魔神化加速 |
| ORC | 鬼人族 | オーク・トロル・ゴーレム | 被ダメージ軽減・壁役 |

## 3. Layer 1 — 部分シナジー（同一種族 2体）

3体中 **2体以上** が同一種族のとき発動。パーティ全体に適用。

| 種族 | 部分シナジー名 | 効果 |
|---|---|---|
| UNDEAD × 2 | 死者の絆 | ターン開始時にパーティ全体 HP +2% 回復 |
| DEMON × 2 | 闇の盟約 | 闇属性ダメージ +10% |
| BEAST × 2 | 野生の俊足 | パーティ全体 SPD +15（AV計算に反映） |
| HUMANOID × 2 | 人の意地 | 会心率 +5% |
| DRAGON × 2 | 竜鱗の共鳴 | 全属性ダメージ加成 +12% |
| ORC × 2 | 鬼の厚皮 | 受けるダメージ -15% |

> **注意**: Layer 2（3体）が発動している場合、Layer 1 は上書きされ **累積しない**。

## 4. Layer 2 — 完全シナジー（同一種族 3体）

3体すべてが同一種族のとき発動。Layer1より強力な効果 + **追加効果** が付与される。

### UNDEAD × 3：「骸軍の不滅陣」

| 効果 | 詳細 |
|---|---|
| HP 回復 | ターン開始時 HP +5%（Layer1 の 2.5倍） |
| **追加: 状態異常耐性** | 毒（POISON）・出血（BLEED）による被ダメージを完全無効化（doc17 §4 の DoT を0にする） |

> **設計意図**: 「死者は毒も病気も受け付けない」世界観。長期戦の周回に最強格。

---

### DEMON × 3：「魔族の悪意」

| 効果 | 詳細 |
|---|---|
| 闇属性ダメージ | +20%（Layer1 の 2倍） |
| **追加: effectHit 強化** | effectHit +25%（状態異常命中率が大幅上昇。doc09 §8 の式に加算） |

> **設計意図**: 状態異常ビルド（effectHit重視）の最高峰。doc17 の WEAKEN / POISON ビルドと相乗。

---

### BEAST × 3：「野獣の本能」

| 効果 | 詳細 |
|---|---|
| SPD 上昇 | +30（全体）。SPD=110 → 140 相当（AV 71→79% に短縮） |
| **追加: エネルギー蓄積** | 毎ターン エネルギー +15（doc09 §5 の通常攻撃 +20 に加算） |

> **設計意図**: 魔神化（Energy=100で発動）を最短で撃てる高速編成。Enrage前に魔神技を決める戦略。

---

### HUMANOID × 3：「英雄の末裔」

| 効果 | 詳細 |
|---|---|
| 会心率 | +10%（Layer1 の 2倍） |
| **追加: 会心ダメージ** | 会心ダメージ +30%（doc09 §7 の critDmg に加算） |

> **設計意図**: 会心特化で単発ダメージを最大化。装備で CRIT_RATE / CRIT_DMG を積む層との相乗。

---

### DRAGON × 3：「天上の覇竜陣」

| 効果 | 詳細 |
|---|---|
| 全属性ダメージ | +25%（Layer1 の 2倍強） |
| **追加: 魔神化加速** | ターン毎の魔神化ゲージ蓄積 +5（通常 +10 → +15/ターン。doc16 §2.1 に加算） |

> **設計意図**: 魔神化を頻繁に発動しながら属性ダメージで押し切る。弱点属性ヒットとの相乗が最大。

---

### ORC × 3：「鉄城の守護陣」

| 効果 | 詳細 |
|---|---|
| 被ダメージ軽減 | -25%（Layer1 の 1.67倍） |
| **追加: 吸収反撃** | 受けたダメージの 10% を HP 回復に変換（タンク編成の生存力をさらに底上げ） |

> **設計意図**: 敵の大技（Enrage）でも即死しない最強防御編成。魔神化の3ターン中に全力攻撃する戦術。

## 5. Layer 3 — クロス共鳴（特定2種族の組み合わせ）

パーティに **特定の2種族が1体以上ずつ** 存在するときに発動。
Layer 1 / Layer 2 とは **独立して加算** される。

2体での「部分シナジー + クロス共鳴」は、3体完全統一の競合となることを意図している。

| 共鳴名 | 発動条件 | 効果 |
|---|---|---|
| **怨念の共鳴** | UNDEAD + DEMON（各1体以上） | 毒・出血の持続ターン +1 / effectHit +10% |
| **天地の覇者** | BEAST + DRAGON（各1体以上） | 戦闘開始時にプレイヤーの AV -20（先制有利）/ 全属性ダメージ +10% |
| **鉄の盟約** | ORC + HUMANOID（各1体以上） | DEF +20% / ATK +8%（物理型の全ラウンド底上げ） |
| **不滅の鎧兵** | UNDEAD + ORC（各1体以上） | 受けるダメージ -10% / ターン開始時 HP +2%（効果を両方享受） |
| **深淵の竜炎** | DEMON + DRAGON（各1体以上） | 炎（FIRE）・闇（DARK）属性ダメージ +20%（この2属性に限定） |
| **俊足剣士団** | BEAST + HUMANOID（各1体以上） | SPD +10 / 会心率 +5%（バランス型で汎用性高） |

### クロス共鳴の重複ルール

1+1+1 の3種族パーティでは、組み合わせの数だけ共鳴が同時発動しうる。

**例: UNDEAD(1) + DEMON(1) + DRAGON(1) の場合**
```
UNDEAD×1 → Layer1 未発動（2体以上必要）
DEMON×1  → Layer1 未発動
DRAGON×1 → Layer1 未発動
怨念の共鳴  (UNDEAD + DEMON) → 発動 ✅
深淵の竜炎  (DEMON + DRAGON) → 発動 ✅
天地の覇者  (BEAST + DRAGON) → 未発動（BEASTなし）
→ 合計: 毒/出血持続+1 / effectHit+10% / 炎・闇ダメージ+20%
```

## 6. 発動判定ルールまとめ

```typescript
// src/logic/TribeSynergySystem.ts

interface SynergyBonus {
  hpRegenPct?: number;       // ターン開始時 HP 回復%
  darkDmgBonus?: number;     // 闇属性ダメージ加成%
  elementDmgBonus?: number;  // 全属性ダメージ加成%
  fireDarkDmgBonus?: number; // 炎・闇限定ダメージ加成%
  spdBonus?: number;         // SPD 加算（flat）
  critRateBonus?: number;    // 会心率加算%
  critDmgBonus?: number;     // 会心ダメージ加算%
  defenseReducePct?: number; // 受けるダメージ軽減%
  effectHitBonus?: number;   // effectHit 加算%
  energyPerTurn?: number;    // ターン毎エネルギー加算
  demonGaugePerTurn?: number;// ターン毎魔神化ゲージ加算
  avBonus?: number;          // 戦闘開始時 AV 加算（負値で先制）
  atkBonus?: number;         // ATK 加算%
  absorbDmgPct?: number;     // 被ダメージ→HP変換%
  ailmentImmune?: ('POISON' | 'BLEED')[]; // 特定状態異常無効
  ailmentDurationBonus?: number;          // 状態異常付与の持続+Nターン
}

function calculatePartyTribeSynergy(monsters: MonsterData[]): SynergyBonus {
  const counts = countTribes(monsters);  // Record<Tribe, number>
  const bonus: SynergyBonus = {};

  // Layer 1 / 2: 同一種族カウント
  for (const [tribe, count] of Object.entries(counts)) {
    if (count >= 3) applyFullSynergy(tribe as Tribe, bonus);
    else if (count >= 2) applyPartialSynergy(tribe as Tribe, bonus);
  }

  // Layer 3: クロス共鳴（各種族が1体以上存在するか）
  const tribes = new Set(Object.keys(counts) as Tribe[]);
  applyCrossResonance(tribes, bonus);

  return bonus;
}
```

## 7. BattleEngine への統合

`calculatePartyTribeSynergy()` はバトル開始時に1度だけ計算し、`BattleState` に保持する。

```typescript
// BattleEngine.ts — バトル初期化時
const synergyBonus = calculatePartyTribeSynergy(monsters);

// ターン開始処理（processStartOfTurn）
if (synergyBonus.hpRegenPct) {
  const regen = Math.floor(player.stats.hp * synergyBonus.hpRegenPct / 100);
  player.currentHp = Math.min(player.stats.hp, player.currentHp + regen);
}
if (synergyBonus.energyPerTurn) {
  player.currentEnergy = Math.min(player.maxEnergy,
    player.currentEnergy + synergyBonus.energyPerTurn);
}
if (synergyBonus.demonGaugePerTurn) {
  this.fillDemonGauge(synergyBonus.demonGaugePerTurn);
}

// ダメージ計算（calculateDamage）に渡す追加補正
// elementDmgBonus / darkDmgBonus / fireDarkDmgBonus → elementBoostMult に加算
// critRateBonus / critDmgBonus → 会心判定に加算

// 状態異常付与（tryApplyAilment）時
// ailmentImmune: 対象種族の DoT を0ダメージとして処理
// ailmentDurationBonus: baseDebuffDuration + bonus として設定

// 戦闘開始時 AV 調整
if (synergyBonus.avBonus) {
  player.currentAV += synergyBonus.avBonus;  // 負値で先制
}
```

## 8. 魔神化との相互作用（doc16 §2.2 との接続）

| シナジー | 魔神化中の扱い |
|---|---|
| BEAST × 3「エネルギー +15/ターン」 | 魔神化中も継続。魔神化解除後の次魔神化が早くなる |
| DRAGON × 3「魔神化ゲージ +5/ターン」 | 魔神化中はゲージ消費（-20/ターン）が優先。解除後に再蓄積開始 |
| UNDEAD × 3「状態異常無効」 | 魔神化中の「全状態異常無効化（doc16 §2.2）」と効果が重複するが、矛盾なし（UNDEAD側は被DoT0、魔神化側は付与試行を飛ばす） |
| ORC × 3「被ダメ -25%・吸収」 | 魔神化中の Glass Cannon（GLASS_CANNON リスク）と組み合わせた場合、`(1 - 0.25) × 2 倍 = 1.5倍` となりリスクを部分的に相殺する（設計上の戦術的選択として許容） |

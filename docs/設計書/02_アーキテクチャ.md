## 3. 主要クラス・サービス責務表

| クラス/サービス | ファイル | 責務 | 依存 |
|---|---|---|---|
| **BattleEngine** | `src/logic/BattleEngine.ts` | ターン制戦闘シミュレーション。`simulateAction()` → `BattleLog[]` を返す | MasterDataService のみ |
| **GameManager** | `src/logic/GameManager.ts` | ゲームループ全体の調整（startStage, processStageResult, soulStone） | Prisma + 全サービス |
| **MasterDataService** | `src/services/MasterDataService.ts` | `src/data/master/*.json` を1回読み込みキャッシュするシングルトン | JSONファイル |
| **JobService** | `src/services/JobService.ts` | 転職処理、レベルアップ、永続パッシブ蓄積 | Prisma (optional), MasterDataService |
| **NecroService** | `src/services/NecroService.ts` | 魂石化、RankUp転生、SoulShard装備 | Prisma |
| **RewardService** | `src/services/RewardService.ts` | EXP計算、ドロップ率計算（LUCK補正） | なし（純粋関数） |
| **useGameStore** | `src/store/useGameStore.ts` | 全クライアント状態管理。`initialize()` でモックデータ投入 | Zustand |

## 6. データフロー（BattleCanvas ↔ BattleEngine）

```
ユーザーがスキルボタンをクリック
  → useGameStore.setActionTrigger({ type: 'MAGIC_SKILL', skillId: 'skill_mage_1' })
  → BattleCanvas の useEffect が actionTrigger を検知
  → BattleEngine.simulateAction('MAGIC_SKILL', targetMonster, 'skill_mage_1') を呼ぶ
  → BattleLog[] を受け取る
  → PixiJS VFX再生 + Framer Motion UI更新
  → useGameStore.addBattleLog(...) でログをストアに追加
```

BattleEngine は Zustand / React に **一切依存しない**。テスト可能な純粋クラスとして設計。

## 7. マスターデータ JSON スキーマ概要

| ファイル | レコード数 | キー構造 |
|---|---|---|
| `jobs.json` | 5 | `{ [jobId]: { name, tier, category, mpCurve, levelBonuses, skills } }` |
| `monsters.json` | 5 | `{ [monsterId]: { name, tribe, cost, stats, resistances } }` |
| `items.json` | 4 | `{ [itemId]: { name, type, rarity, stats, specialEffect? } }` |
| `stages.json` | 3 | `{ [stageId]: { name, area, waves, enemies[][], rewards, position } }` |
| `skills.json` | 11+ | `{ [skillId]: { id, name, mpCost, power, type, element, attackType, targetType, effectKey, description } }` |

詳細スキーマは **[04_データモデル.md](04_データモデル.md)** を参照。

## 8. 新機能追加時の変更箇所ガイド

### 新しい職業を追加する場合
1. `src/data/master/jobs.json` に職業定義追加
2. `src/data/master/skills.json` にスキル追加（必要な場合）
3. `src/types/game.ts` — 型に影響がある場合のみ変更
4. `src/components/legion/LegionHub.tsx` — JOBカラーマップ（`JOB` オブジェクト）に追加

### 新しいタブ画面を追加する場合
1. `src/store/useGameStore.ts` — `currentTab` の型に追加
2. `src/components/layout/BottomNavBar.tsx` — `TABS` 配列に追加
3. `src/app/page.tsx` — `renderMainContent()` の switch に追加
4. 新コンポーネントを `src/components/<feature>/` に作成

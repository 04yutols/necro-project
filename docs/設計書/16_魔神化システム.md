## 1. コアコンセプト

主人公アルドは表向き「勇者の遺産（各職業）」を装備して戦うが、
ソウルゲージがMAXになった時、根底にある**「魔族の力（死霊術師）」**をその職業に注ぎ込み、
光の力を反転・暴走させる。これを「**魔神化（Demonization）**」と呼ぶ。

> **世界観補足**: 「剣士の魔神化」は剣聖が闇に堕ちた姿であり、「メイジの魔神化」は秩序の魔法が
> 死霊の呪いに侵食された姿。職業の「正の力」が「負の力」に反転する瞬間がビジュアルで表現される。

## 2. 基本システム仕様（共通）

### 2.1 実装済みステート（`useGameStore.ts` / `BattleCanvas.tsx`）

```typescript
demonGauge:   number;   // 0〜100
isDemonMode:  boolean;
demonActionsRemaining: number; // 主人公の残り行動数
demonUltimateUsed: boolean;    // 魔神技の使用済みフラグ
demonFormJobId: string | null; // 現在職業に対応するフォームID

fillDemonGauge(amount: number)  // 0〜100にクランプ
startDemonMode(jobId?: string)  // 発動。ゲージを0に消費
consumeDemonAction()            // 主人公行動ごとに残数を減らす
markDemonUltimateUsed()         // 魔神技を1回使用済みにする
endDemonMode()                  // 強制解除
```

| 要素 | 実装仕様 |
|---|---|
| ゲージ蓄積 | 通常行動+10、WAVEクリア+18、防壁破壊+30 |
| 発動条件 | `demonGauge >= 100 && !isDemonMode` |
| 発動 | ゲージを0に消費し、敵ターン中なら敵行動タイマーをキャンセルして自ターンへ割り込む |
| 持続 | 主人公が3回行動するまで |
| 消費 | 発動中の毎ターン-20消費は廃止。`ENERGY_DRAIN` リスク職のみ追加減衰 |
| 効果 | 職業別フォームの Effect A/B、状態異常無効、職業別VFX、1回限りの魔神技 |
| 解除 | 3回行動経過 または `toggleDemonMode()` / `endDemonMode()` |

### 2.2 共通仕様

旧仕様の「一律+20%」「魔神専用スキル3種」は廃止し、**装備中の職業ごとに固有の魔神化フォーム**を適用する。

| 追加要素 | 詳細 |
|---|---|
| **発動（絶対割り込み）** | ゲージMAX時に任意発動。発動瞬間にプレイヤーのAV（行動値）= 0 になり、敵の行動をキャンセルして最前列に割り込む |
| **持続** | 主人公が**3回行動するまで**（ターン数ではなく行動数基準）|
| **状態異常無効** | 魔神化中、全ての状態異常・デバフを完全無効化 |
| **魔神技（Ultimate Skill）** | 魔神化中に1度だけ使用可能な専用の奥義 |
| **Effect A（基礎昇華）** | 常時発動する暴力的なステータスバフ |
| **Effect B（深淵の理）** | 職業・Tierに応じたギミックと代償（Tier2は強烈なリスク必須） |

## 3. Tier別 魔神化「尖り（Peaky）」設計ルール

### Tier 1（初期職: 剣士・盗賊・メイジ・闇術師等）

| 項目 | ルール |
|---|---|
| **設計方針** | 分かりやすく強力。純粋なステータスバフ＋扱いやすいギミック |
| **リスク** | デメリットは設定しなくてよい |
| **魔神技** | 単純な大ダメージ、汎用性の高い自己バフ、フィールド生成 |
| **バランス感** | 初見でも「強い！」と感じられる爽快感を優先 |

### Tier 2（上位職: ベルセルク・アークメイジ等）

| 項目 | ルール |
|---|---|
| **設計方針** | 前提の2つの下位職の要素を**悪魔合体**させる |
| **リスク（必須）** | 強烈なクセ（代償）をEffect Bに必ず組み込む |
| **魔神技** | 絶大ダメージ＋戦闘ローテーションを有利にする**持続効果を必ず残す** |
| **禁止事項** | 単純な火力インフレ（HP1で致死ダメージ等）。ターン制RPGの戦略を破壊するため厳禁 |

**Tier 2 で推奨されるリスクパターン**:

| リスク名 | 効果 | パーティへの影響 |
|---|---|---|
| **自傷** | 攻撃するたびに味方全体HPを削る | ヒーラー必須化 |
| **MP過剰消費** | 魔法が連射できるがエネルギー消費数倍 | バッテリー役必須化 |
| **紙装甲** | 火力青天井だが被ダメ 2〜3倍 | タンク・挑発キャラ必須化 |
| **セットアップ依存** | 基礎火力ゼロ＋敵のデバフ数だけ乗算 | デバフコンボ必須化 |

### 4.1 アークメイジ（Tier 2）の魔神化

## 5. 魔神化フォームのマスターデータ構造（実装済み）

```typescript
// src/data/master/demonForms.json — 職業ごとのフォーム定義
interface DemonForm {
  jobId: string;                  // 対応する職業ID（"berserker" 等）
  formName: string;               // 魔神化名（例: "血河の狂戦士"）
  tier: 1 | 2;
  concept: string;
  effectA: {
    descJa: string;               // 表示用テキスト
    statBoosts: Partial<Record<keyof BaseStats, number>>;  // 数値バフ
    flags?: string[];             // 特殊フラグ（例: "IGNORE_RESISTANCE"）
  };
  effectB: {
    descJa: string;
    riskType: 'SELF_DAMAGE' | 'ENERGY_DRAIN' | 'GLASS_CANNON' | 'SETUP_DEPENDENT' | null;
    riskValue?: number;           // Tier1はnull
    onAttackEffect?: string;      // 攻撃時トリガーのフラグ（例: "TRIPLE_HIT"）
  };
  ultimateSkill: {
    nameJa: string;
    damage: {
      power: number;              // SkillData.power と同形式
      element: ElementType;
      targetType: 'SINGLE' | 'ALL';
      attackType?: SkillAttackType;
      flags?: string[];           // 例: "IGNORE_DEF"
    };
    lingering: {
      type: 'FIELD' | 'PARTY_BUFF' | 'ENEMY_DEBUFF';
      descJa: string;
      duration: number;           // ターン数（-1=バトル終了まで）
    };
  };
  visual?: {
    color: string;
    soft: string;
    icon: string;
  };
}
```

## 6. バトル統合（実装メモ）

`src/logic/DemonizationSystem.ts` は UI に依存しない純ロジックとして、以下を提供する。

| 関数 | 役割 |
|---|---|
| `canActivateDemonMode()` | ゲージMAXかつ未発動かを判定 |
| `activateDemonMode()` | ゲージを0に消費し、3行動・魔神技未使用のランタイム状態を返す |
| `consumeDemonAction()` | 主人公行動ごとに残り行動数を減らし、0で解除 |
| `getDemonDamageMultiplier()` | Effect A の火力補正を攻撃種別ごとに解決 |
| `getDemonActionHitCount()` | `TRIPLE_HIT` / `MIRAGE_CHAIN` などの追撃回数を解決 |
| `getDemonIncomingDamageMultiplier()` | `GLASS_CANNON` の被ダメージ倍率を解決 |
| `isDemonStatusImmune()` | 状態異常・デバフ無効状態を判定 |

`BattleCanvas.tsx` では、敵ターン中の発動時に `enemyTurnSerialRef` を進めて予約済み敵行動を無効化し、即座に `playerTurn` へ戻すことで絶対割り込みを表現する。
魔神技は `demonForms.json` から動的に生成し、使用後は同一魔神化中に再使用できない。

## 7. AI生成プロンプトフォーマット（設計ガイド）

新しい職業の魔神化を設計する際は、エージェントスキル `/design-demonform` を使用する。
スキルの定義は `/Users/yuto/.claude/agents/demonization-designer.md` を参照。

手動でプロンプトを送る場合は以下のフォーマットを使用すること:

```
# Instruction
あなたはAAA級ダークファンタジーRPG『ネクロマンスブレイブ』の専属バトルプランナーです。
ガイドラインの「Tier別：魔神化の尖り設計ルール」に従い、以下の職業に対する
【魔神化フォーム】と【魔神技】を設計し、指定のMarkdownフォーマットで出力してください。

# Target Job
- 職業名: [例：アサシン]
- Tier: [1 or 2]
- 職業の特徴: [例：手数と回避に特化。盗賊と剣士の派生]

# Output Format (Markdown)
### 職業名: [職業名] の魔神化
- **魔神化名:** 『[中二病的なダークファンタジー感のある二つ名]』
- **コンセプト:** [プレイスタイルとリスクの簡潔な説明]
- **Effect A (基礎昇華):** [基礎ステータスの暴力的な向上]

- **魔神技 (Ultimate Skill): 『[技名]』**
  - **演出&効果:** [ダメージの性質]
  - **残留効果:** [その後の戦闘を有利にするフィールド・バフ・デバフ]
```

## 6. BattleCanvas — 責務分離

| 担当 | 技術 | 詳細 |
|---|---|---|
| VFX | PixiJS | 属性別エフェクト（炎/水/雷/土/風）、ダメージフロート |
| UI演出 | Framer Motion | スクリーンシェイク、スキルボタン、WAVEトランジション |
| 状態管理 | Zustand + useGameStore | `actionTrigger` 経由で BattleEngine を疎結合に呼び出し |

### 戦闘状態マシン（BattleCanvas 内部）

```
'playerTurn' → ユーザーが行動選択
  → 'animating': VFX再生（920ms〜）
  → endPlayerTurn(): 全敵が死亡？→ resolveWaveClear() / 生存→ runEnemyTurn()

'enemyTurn': 各敵が攻撃（stagger: 0.55×speedMs）
  → 'playerTurn' に戻る

'waveTransition': 次WAVE準備（1200ms delay）
  → 全WAVEクリア: ResultScreen 表示

'demonized': demonTurns=3 / 専用スキル使用可
  → 各ターン demonTurns-- → soul -20
  → demonTurns=0 で通常モードに戻る
```

## 8. iOS Safari 安全ルール

**最重要**: `motion.div` と `overflow: hidden` を同一要素に置かない。

```tsx
// ❌ 危険（iOS SafariでクリッピングがFF）
<motion.div animate={{ y: 0 }} className="absolute inset-0 overflow-hidden">

// ✅ 安全（2層に分離）
<motion.div animate={{ y: 0 }} style={{ position: 'absolute', inset: 0 }}>
  {/* overflow は別の div に */}
  <div className="absolute inset-0 overflow-hidden flex flex-col">
```

**理由**: Framer Motion はアニメーション完了後も `transform: translateY(0px)` を残す（`transform: none` にしない）。iOS Safari は `overflow:hidden` + CSS transform が同一要素にあると絶対配置の子要素をクリップしない。

### レイアウト必須パターン

```tsx
// 固定高さのセクション
<div className="shrink-0" style={{ height: 44 }}>ヘッダー</div>

// 可変高さのセクション（残り全部）
<div className="flex-1 min-h-0 overflow-hidden">コンテンツ</div>

// フッター
<div className="shrink-0">フッター</div>
```

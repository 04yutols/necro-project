## 1. 設計哲学 — 原神 / Star Rail レベルのオンボーディング基準

### 1.1 3つの原則

| 原則 | 原神 / Star Rail の実現例 | 本作への適用 |
|---|---|---|
| **流れを止めない** | 初戦闘はカットシーン→即戦闘。チュートリアルがゲームの一部として融合 | プロローグシーン終了直後に第1戦が始まる。戦闘中にハイライトオーバーレイで教える |
| **1度に1つ** | スキルはWAVE2まで使えない。エネルギーゲージは2日目まで登場しない | バトル→マップ→編成→転職→残滓→魔神化の順でロック解除。1セッション1概念 |
| **2周目は快適に** | 既読スキップ / チュートリアルスキップが常に利用可能 | `tutorialCompleted` フラグ ON で全チュートリアルを省略できる設定ページへの導線 |

### 1.2 「教えない」リスト

ハクスラゲームの熟練者は説明過多を嫌う。以下は**意図的に教えない**:

| 機能 | 理由 |
|---|---|
| クリティカル確率・クリティカルダメージの具体式 | 探索で自然に気づく。数値をUI上で確認可能にするだけで十分 |
| 属性相性の全表 | 第1章は属性に偏りのない敵を配置。プレイヤーが自分で試して気づく |
| 残滓スコアの加重計算式 | §12（深淵の残滓UIUX）に「スコア計算は見せる」設計がある。式の説明は不要 |
| クロス共鳴（種族シナジーの上位層）| 基本シナジーを覚えた後、プレイヤーが自分で発見する喜びとして保持 |
| 転職パッシブの蓄積仕組み | 転職画面のツールチップで確認できる。チュートリアルで覚えさせる必要なし |

### 1.3 第1章スコープ方針

**「第1章前半 = チュートリアル専用コンテンツ」ではない**。

第1章は完全にストーリーコンテンツとして設計し、チュートリアルはその進行中に**後乗り**で挿入する。
- プレイヤーはストーリーを楽しみながら操作を覚える
- チュートリアル完了後に既読スキップが有効化されるため、2周目から邪魔にならない

### 4.2 スポットライトコンポーネント

```typescript
// src/components/tutorial/TutorialSpotlight.tsx

interface SpotlightStep {
  targetId: string;       // DOM要素の id 属性 (例: "energy-gauge")
  title: string;          // 「エネルギーゲージ」
  body: string;           // 「スキルを使うと消費し、通常攻撃で回復する」
  position: 'above' | 'below' | 'left' | 'right';
  onDismiss?: () => void; // タップで次のステップへ
}

function TutorialSpotlight({ step, onDismiss }: { step: SpotlightStep; onDismiss: () => void }) {
  // 1. 対象要素の getBoundingClientRect() を取得
  // 2. 全画面暗転レイヤー (z: 9000, rgba(0,0,0,0.72))
  // 3. 対象位置に合わせた円形クリップ (SVG clipPath + clip-rule:evenodd)
  // 4. バブルテキスト (Void Purple border, framer-motion slideIn)
  // 5. 「次へ」ボタン または 画面任意タップで onDismiss()

```

## 5. スキップ設計

### 5.1 スキップ可否マトリクス

| チュートリアル | 初回 | 2周目以降 | スキップ後の処理 |
|---|---|---|---|
| プロローグシーン（PROLOGUE_00〜03） | スキップ不可 | スキップ可（タップ連打 or スキップボタン） | ストーリーフラグだけ立てて続行 |

| バブルヒント | 画面外タップで消せる | 表示されない | — |
| スポットライト | 画面任意タップで次へ | 表示されない | — |

### 5.2 全スキップ設定

設定画面（24_設定画面設計）に「チュートリアルをリセット」オプションを追加する。
- **ON**: 次回起動時に全チュートリアルを再生
- **OFF（デフォルト）**: `tutorialCompleted` フラグが ON なら一切表示しない

## 6. ストア設計

```typescript
// src/store/useTutorialStore.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface TutorialState {
  // フェーズ管理
  completedPhases: TutorialPhase[];       // 完了済みフェーズID配列
  activePhase: TutorialPhase | null;      // 現在進行中のフェーズ
  activeStepIndex: number;               // フェーズ内のステップ番号
  tutorialCompleted: boolean;            // 全フェーズ完了フラグ

  // ヒント管理
  viewedHints: string[];                 // バブルヒントの既読ID

  // アクション
  startPhase: (phase: TutorialPhase) => void;
  nextStep: () => void;
  skipPhase: () => void;
  markHintViewed: (hintId: string) => void;
  completeTutorial: () => void;
  resetTutorial: () => void;
}

type TutorialPhase =
  | 'BATTLE_BASICS'
  | 'NECRO_LAB'
  | 'PARTY_FORMATION'
  | 'JOB_CHANGE'
  | 'ABYSSAL_RESIDUE'
  | 'DEMONIZATION';

export const useTutorialStore = create<TutorialState>()(
  persist(
    (set, get) => ({
      completedPhases: [],
      activePhase: null,
      activeStepIndex: 0,
      tutorialCompleted: false,
      viewedHints: [],

      startPhase: (phase) =>
        set({ activePhase: phase, activeStepIndex: 0 }),

      nextStep: () => {
        const { activePhase, activeStepIndex } = get();
        if (!activePhase) return;
        const steps = PHASE_STEPS[activePhase];
        if (activeStepIndex + 1 >= steps.length) {
          set(s => ({
            completedPhases: [...s.completedPhases, activePhase],
            activePhase: null,
            activeStepIndex: 0,
          }));
        } else {
          set({ activeStepIndex: activeStepIndex + 1 });
        }
      },

      skipPhase: () => {
        const { activePhase } = get();
        if (!activePhase) return;
        set(s => ({
          completedPhases: [...s.completedPhases, activePhase],
          activePhase: null,
          activeStepIndex: 0,
        }));
      },

      markHintViewed: (hintId) =>
        set(s => ({ viewedHints: [...s.viewedHints, hintId] })),

      completeTutorial: () => set({ tutorialCompleted: true }),

      resetTutorial: () =>
        set({
          completedPhases: [],
          activePhase: null,
          activeStepIndex: 0,
          tutorialCompleted: false,
          viewedHints: [],
        }),
    }),
    { name: 'tutorial-store' }
  )
);
```

## 7. トリガー設計

### 7.1 `useTutorialTrigger` フック

```typescript
// src/hooks/useTutorialTrigger.ts
import { useEffect } from 'react';
import { useGameStore } from '../store/useGameStore';
import { useTutorialStore } from '../store/useTutorialStore';

export function useTutorialTrigger() {
  const clearedStages = useGameStore(s => s.clearedStages);
  const inventoryMonsters = useGameStore(s => s.inventoryMonsters);
  const { completedPhases, tutorialCompleted, startPhase } = useTutorialStore();

  useEffect(() => {
    if (tutorialCompleted) return;

    // PHASE 1: 初回バトルクリア後 → 死霊術フェーズ
    if (
      clearedStages.includes('tutorial_battle_01') &&
      !completedPhases.includes('NECRO_LAB')
    ) {
      startPhase('NECRO_LAB');
    }

    // PHASE 3: モンスター2体以上 + 朽ちた回廊クリア後 → 編成フェーズ
    if (
      inventoryMonsters.length >= 2 &&
      clearedStages.includes('tutorial_battle_02') &&
      !completedPhases.includes('PARTY_FORMATION')
    ) {
      startPhase('PARTY_FORMATION');
    }

    // PHASE 4: モンスター3体以上 → 転職フェーズ
    if (
      inventoryMonsters.length >= 3 &&
      completedPhases.includes('PARTY_FORMATION') &&
      !completedPhases.includes('JOB_CHANGE')
    ) {
      startPhase('JOB_CHANGE');
    }

    // 全フェーズ完了チェック
    const allPhases: TutorialPhase[] = [
      'BATTLE_BASICS', 'NECRO_LAB', 'PARTY_FORMATION',
      'JOB_CHANGE', 'ABYSSAL_RESIDUE', 'DEMONIZATION'
    ];
    if (allPhases.every(p => completedPhases.includes(p))) {
      useTutorialStore.getState().completeTutorial();
    }
  }, [clearedStages, inventoryMonsters, completedPhases]);
}
```

### 8.1 `TutorialOrchestrator`

`src/app/page.tsx` の最上位（StoryOrchestrator の直上）に配置し、`z-index: 9500`（StoryOrchestrator より上）で常時マウント。

```typescript
// src/components/tutorial/TutorialOrchestrator.tsx
export function TutorialOrchestrator() {
  const { activePhase, activeStepIndex, tutorialCompleted } = useTutorialStore();
  useTutorialTrigger();  // 副作用: ゲーム状態を監視してフェーズを発火
```

## 10. ビジュアル仕様

### 10.1 スポットライトのフレームスペック

```
f0          : 暗転レイヤー フェードイン (opacity 0→0.72, 200ms ease-out)
f12 (200ms) : スポットライト円 スケールイン (scale 2→1, 300ms spring)
f30 (500ms) : バブルテキスト スライドイン (y +8→0, opacity 0→1, 250ms ease-out)
              「次へ」ボタン フェードイン (150ms delay)
DISMISS     : バブルテキスト フェードアウト (100ms)
              暗転レイヤー フェードアウト (200ms)
              次ステップへ
```

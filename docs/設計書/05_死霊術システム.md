## 1. 育成軸の全体像

死霊術システムは **職業システムと完全に独立した第2の育成軸**。  
職業はプレイヤー自身の能力、死霊術はプレイヤーが率いる「軍団」の能力。

```
職業システム (GDD-004)        死霊術システム (GDD-005)
───────────────             ─────────────────────
主人公自身のステータス        軍団のコスト上限・ステータス補正
12種の職業（Tier1/Tier2）    レベル(Lv.99) + ランク(Rank.10)
転職してもパッシブは残る      RankUpで転生リセット→爆発的成長
```

## 2. 死霊術ステータス

| フィールド | 型 | 初期値 | 説明 |
|---|---|---|---|
| `necroLevel` | Int | 1 | 1〜99。サクサク上がる |
| `necroRank` | Int | 1 | 1〜10。転生（試練クリア必須） |
| `necroMaxCost` | Int | 10 | パーティに配備できる合計コスト上限 |
| `necroBaseStatsBonus` | Float | 1.0 | 軍団の基礎ステータス倍率 |

クライアント側では `NecroStatus` 型として保持:
```typescript
interface NecroStatus {
  level: number;       // max 99
  rank: number;        // max 10
  maxCost: number;
  baseStatsBonus: number;
}
```

## 3. Lv.99 → RankUp 転生システム

```
条件: necroLevel >= 99 AND 試練クリア済み (isTrialCompleted=true)

転生後の変化:
  necroLevel    → 1 (リセット)
  necroRank     → Math.min(10, necroRank + 1)
  necroMaxCost  → necroMaxCost + 5   (例: 10 → 15 → 20 → ...)
  necroBaseStatsBonus → necroBaseStatsBonus + 0.5  (例: 1.0 → 1.5 → 2.0 → ...)
```

**設計意図**: Lv.1〜99は「サクサク上がる」チュートリアル的成長。Rank UPが本当の強化。Rank10になると `necroMaxCost=55`, `necroBaseStatsBonus=5.5` まで到達。

実装: `NecroService.performRankUp(characterId, isTrialCompleted)`

## 4. 3枠コスト制の軍団編成

```
party: (MonsterData | null)[]  // 常に3要素の配列

コスト検証:
  totalCost = party.filter(Boolean).reduce((sum, m) => sum + m.cost, 0)
  totalCost <= necroMaxCost でなければ配備不可

エラー: "コスト超過です" (NecroService内) / UIは警告ハイライト表示
```

### モンスターのコスト感

| 種別 | cost | 例 |
|---|---|---|
| 弱小モンスター | 1 | Goblin, Skeleton, Zombie |
| 中堅モンスター | 2 | Imp |
| ボス格モンスター | 3 | Goblin King |

初期 `maxCost=10` の場合: cost1×3=3（余裕）/ cost3+cost3+cost3=9（ほぼ上限）など様々な組み合わせ。

## 5. 魂石化（Soul Stoning）フロー

### 3段階処理

```
[1] UI: 確認ダイアログ表示 ("このモンスターを魂石化しますか？")
         ↓
[2] Server: NecroService.createSoulShard(monsterId) を呼ぶ
     └─ Prisma $transaction:
          monster.atk × 0.1 → atkBonus
          monster.effectHit × 0.1 → elementDmgBoost
          tribe → specialAbility を deriveSpecialAbility() で決定
          SoulShard を DB に create
          Monster を DB から delete  ← 取り消し不可
     └─ SoulShardData を返す
         ↓
[3] Client: Zustand ストア更新
     inventoryMonsters から削除
     soulShards に追加
     party から該当モンスターを null に
```

### SoulShard 効果値計算

```typescript
atkBonus  = Math.floor(monster.atk  * 0.1)
elementDmgBoost = Math.floor(monster.effectHit * 0.1)
```

例: ATK=50のモンスター → SoulShard の atkBonus=5。effectHit=80 のモンスター → 属性ダメージ加成 +8%。

## 6. 種族別 specialAbility

```typescript
private deriveSpecialAbility(tribe: Tribe): string | undefined {
  switch (tribe) {
    case 'UNDEAD': return "REGENERATE_SOUL";  // HP微量回復
    case 'DEMON':  return "BANE_OF_LIGHT";    // 光属性耐性持ち敵へのダメージ増
    default:       return undefined;           // BEAST / HUMANOID は未設定
  }
}
```

## 9. 霊核（SpiritCore）仕様

```typescript
interface SpiritCoreData {
  id: string;
  name: string;
  element?: ElementType;    // モンスターに属性を付与
  skillChangeId?: string;   // 使用スキルを差し替え
  atkMultiplier: number;    // ATK倍率（デフォルト1.0）
}
```

モンスターの `spiritCore` に装着することで:
- `element` → モンスターの攻撃に属性付与（耐性計算に適用）
- `skillChangeId` → モンスターが使うスキルを変更
- `atkMultiplier` → 追撃ダメージに乗算

## 10. 職業との連携（GDD-005 × GDD-004）

| 状態 | 効果 |
|---|---|
| 通常時（currentJobId に応じて） | 軍団全体にジョブ依存パッシブバフ（設計段階） |
| `isAwakened == true` | 軍団の ATK を **1.5倍** にして BattleEngine に渡す |

実装箇所: `BattleEngine.processMonsterActions()`:
```typescript
if (player.isAwakened) {
  monsterStats.atk  = Math.floor(monsterStats.atk  * 1.5);
}
```

## 1. 設計哲学 — 原神 / Star Rail レベルの物語 UX 基準

### 1.1 体験設計の4原則

| 原則 | 原神の実現例 | 本作への適用 |
|---|---|---|
| **感情への没入** | 章タイトルカード + フルスクリーン演出で「別世界に来た」感覚 | 黒帯レターボックス + Void Purple グロー + Gothic 書体で中世ゴシックの世界観を強調 |
| **テンポの主導権をプレイヤーへ** | タップで1文字ずつ進む / スキップ常時可能 | オート進行トグル + 既読シーン即スキップ + ページめくり感 |
| **アーカイブで安心感** | 星穹列車の「記憶」 / 原神の「伝説任務ログ」 | 既読シーンを LOGS タブからいつでも再閲覧 |
| **環境がセリフを語る** | 探索中のモノローグ・BGM変化・エフェクト | ステージ入場前の環境描写テキスト + PixiJS 背景パーティクル変化 |

### 1.2 システム全体像

```
ストーリーデータ (JSON)
  └─ src/data/story/
        ├── act1_prologue.json        ← 第0章: プロローグ
        ├── act1_ch1_royal_capital.json ← 第1章: 亡国の王都
        ├── act1_ch2_phantom_city.json  ← 第2章: 幽霊都市
        └── ...

トリガー判定
  └─ useStoryTrigger() フック
        ├── ステージクリア後 → 対応シーンを発火
        ├── ボス撃破後 → 章後エピローグ発火
        ├── 初回ログイン → プロローグ発火
        └── エリア解放時 → 解放演出発火

表示レイヤー
  └─ StoryOrchestrator（src/app/page.tsx 上位に配置）
        ├── DialogueScene        — 会話（複数キャラ）
        ├── MonologueOverlay     — 独白・内面描写
        ├── EnvironmentCaption   — 環境描写テキスト
        ├── ChapterTitleCard     — 章タイトル演出
        └── StoryArchive         — 再閲覧アーカイブ
```

```typescript
// src/types/story.ts

/** シーンの種別 */
export type SceneType =
  | 'DIALOGUE'       // キャラ立ち絵つき会話（複数発言者可）
  | 'MONOLOGUE'      // 主人公独白（全画面暗転 + テキスト）
  | 'ENVIRONMENT'    // 環境描写（BGM + テキスト、キャラなし）
  | 'CHAPTER_TITLE'  // 章タイトルカード
  | 'CHOICE'         // 選択肢（Phase 2 実装予定）

/** トリガー条件 */
export type SceneTrigger =
  | { type: 'GAME_START' }                        // 初回起動
  | { type: 'STAGE_CLEAR'; stageId: string }      // ステージクリア直後
  | { type: 'STAGE_ENTER'; stageId: string }      // ステージ入場直前
  | { type: 'AREA_UNLOCK'; areaId: string }       // エリア解放時
  | { type: 'BOSS_CLEAR'; bossStageId: string }   // ボス撃破直後
  | { type: 'MANUAL'; sceneId: string }           // アーカイブから手動再生

/** キャラクター表示 */
export interface CharacterPortrait {
  characterId: string;       // 'aldo' | 'line' | 'demon_king' | 'narrator'
  position: 'LEFT' | 'CENTER' | 'RIGHT';
  expression: string;        // 'default' | 'sad' | 'angry' | 'determined' | 'smile'
  isDimmed?: boolean;        // 話していない時は暗くする（原神方式）
}

/** 1発言単位 */
export interface DialogueLine {
  speaker: string | null;    // null = ナレーター（名前表示なし）
  speakerJa?: string;        // 表示名（Jaパネルに表示）
  text: string;              // メインテキスト（日本語）
  textEn?: string;           // 英語サブタイトル（任意）
  portraits?: CharacterPortrait[];  // この発言中の立ち絵配置
  expression?: string;       // 発言者の表情変化
  bgm?: string;              // BGM変更（ファイルパス or BGM_ID）
  vfx?: string;              // VFX発火キー（21_VFX参照）
  pauseAfter?: number;       // 自動進行時の追加ウェイト ms
  skipIf?: string[];         // このフラグが立っていればスキップ
}

/** 1シーン */
export interface StoryScene {
  id: string;                // ユニークID 例: 'PROLOGUE_00'
  type: SceneType;
  title?: string;            // 章タイトルカード用
  trigger: SceneTrigger;
  background?: string;       // 背景画像パス or 'BLUR_MAP' | 'DARK' | 'NONE'
  lines: DialogueLine[];
  onComplete?: {             // シーン終了後アクション
    unlockArea?: string;
    setFlag?: string;
    navigateTo?: string;     // 'MAP' | 'HOME' | etc.
  };
  isSkippable: boolean;      // false = プロローグ等はスキップ不可
  archiveTitle: string;      // アーカイブに表示するタイトル
  archiveChapter: number;    // 1 = 第1章
}
```

## 3. UIコンポーネント設計

### 3.1 `StoryOrchestrator` — 最上位制御コンポーネント

```typescript
// src/components/story/StoryOrchestrator.tsx

/**
 * page.tsx の最上位に配置し、ゲーム全体のシーン表示を制御する。
 * 子コンテンツ（ゲーム本体）の上に absolute で重なる。
 */
export function StoryOrchestrator({ children }: { children: React.ReactNode }) {
  const { activeScene, dismissScene } = useStoryStore();
  return (
    <>
      {children}
      <AnimatePresence>
        {activeScene?.type === 'CHAPTER_TITLE' && <ChapterTitleCard scene={activeScene} onDone={dismissScene} />}
        {activeScene?.type === 'DIALOGUE'      && <DialogueScene    scene={activeScene} onDone={dismissScene} />}
        {activeScene?.type === 'MONOLOGUE'     && <MonologueOverlay scene={activeScene} onDone={dismissScene} />}
        {activeScene?.type === 'ENVIRONMENT'   && <EnvironmentCaption scene={activeScene} onDone={dismissScene} />}
      </AnimatePresence>
    </>
  );
}
```

## 4. トリガーシステム

### 4.1 `useStoryTrigger()` フック

```typescript
// src/hooks/useStoryTrigger.ts

/**
 * ゲームイベント（ステージクリア・エリア解放等）を監視し、
 * 対応するストーリーシーンをキューに積む。
 */
export function useStoryTrigger() {
  const { clearedStages, currentTab } = useGameStore();
  const { enqueueScene, viewedScenes } = useStoryStore();

  // ステージクリア監視
  useEffect(() => {
    const lastCleared = clearedStages[clearedStages.length - 1];
    if (!lastCleared) return;
    const scenes = SCENE_REGISTRY.filter(s =>
      s.trigger.type === 'STAGE_CLEAR' &&
      s.trigger.stageId === lastCleared &&
      !viewedScenes.has(s.id)
    );
    scenes.forEach(s => enqueueScene(s.id));
  }, [clearedStages]);

  // 初回起動監視
  useEffect(() => {
    if (!viewedScenes.has('PROLOGUE_00')) {
      enqueueScene('PROLOGUE_00');
    }
  }, []);
}
```

### 4.2 発火タイミング詳細

| イベント | 表示タイミング | 優先度 |
|---|---|---|
| 初回起動 | HomeHero 表示直前 | 最高 |
| ステージ入場 | バトル開始アニメーション直前 | 高 |
| ステージクリア | ResultScreen 表示直前 | 高 |
| ボス撃破 | ResultScreen の前に専用エピローグ挿入 | 最高 |
| エリア解放 | AreaMap の解放アニメーション前 | 中 |
| 章クリア | ChapterTitleCard（次章タイトル）を自動表示 | 最高 |

```typescript
// 複数シーンが同時に積まれた場合（例: ボス撃破 → 章クリア）

}
```

```typescript
      enqueueScene: (sceneId) => {
        const { sceneQueue, activeScene, viewedScenes } = get();
        if (viewedScenes.has(sceneId)) return; // 既読はスキップ
        if (!activeScene) {
          const scene = findScene(sceneId);
          set({ activeScene: scene });
        } else {
          set({ sceneQueue: [...sceneQueue, sceneId] });
        }
      },

      dismissScene: () => {
        const { sceneQueue, activeScene } = get();
        get().markViewed(activeScene?.id ?? '');
        if (activeScene?.onComplete?.setFlag) {
          get().setFlag(activeScene.onComplete.setFlag);
        }
        const [next, ...rest] = sceneQueue;
        set({
          activeScene: next ? findScene(next) : null,
          sceneQueue: rest,
        });
      },

      markViewed: (sceneId) => set(s => ({
        viewedScenes: new Set([...s.viewedScenes, sceneId])
      })),

      setFlag: (flagKey) => set(s => ({
        storyFlags: { ...s.storyFlags, [flagKey]: true }
      })),
    }),
    { name: 'story-store', partialize: (s) => ({
        viewedScenes: [...s.viewedScenes],
        storyFlags: s.storyFlags,
      })
    }
  )
);
```

## 8. 操作仕様 / UX ルール

### 8.1 進行操作

| 操作 | 動作 |
|---|---|
| 画面タップ（テキスト表示中） | 残りテキストを即時全表示 |
| 画面タップ（テキスト全表示後） | 次の発言へ進む |
| 右上「SKIP ▶▶」ボタン | シーン全体をスキップ（既読の場合のみ表示）|
| 長押し（0.8s） | オート進行モードのON/OFF切り替え |

### 8.2 オート進行モード

```
オートON時:
  全文表示後 → 1.5s 待機 → 自動で次の発言へ
  句読点後: +0.5s の追加ウェイト
  シーン最終発言後: 3s 待機 → シーン終了

オート速度:
  FAST: 20ms/文字, 0.8s 発言間
  NORMAL: 40ms/文字, 1.5s 発言間（デフォルト）
```

### 8.3 スキップルール

```typescript
// 未読シーン: SKIPボタン非表示（強制的に読む）
//   ※ ただし isSkippable: false のシーンのみ
// 既読シーン: SKIPボタン表示 → タップで即座に dismissScene()
// 既読かつキューに複数: 「全スキップ」ボタンも表示

const canSkip = viewedScenes.has(activeScene.id) || activeScene.isSkippable;
```

### 8.4 バトル中の割り込み禁止

```typescript
// STAGE_ENTER トリガーはバトル開始 " 前 " に表示
// STAGE_CLEAR トリガーはバトル終了 " 後 "・ResultScreen 表示 " 前 " に表示
// currentTab === 'BATTLE' の間はシーンキューを保留

useEffect(() => {
  if (currentTab !== 'BATTLE' && sceneQueue.length > 0) {
    dequeueNextScene();
  }
}, [currentTab]);
```

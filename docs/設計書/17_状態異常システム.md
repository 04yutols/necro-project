## 1. システム概要

状態異常は「発動判定 → 効果適用 → ターン経過管理」の3フェーズで処理される。
核となる発動式は `09_ステータスシステム §8` で定義済みの以下を使用する。

```
発動確率 = baseDebuffRate × (1 + effectHit / 100) × (1 - target.effectRes / 100)
```

**ゲームデザイン上の位置付け**:

| 側面 | 内容 |
|---|---|
| 戦略軸 | effectHit を積む「状態異常ビルド」を剣士・盗賊型の差別化として機能させる |
| リスク | 敵が付与する状態異常でプレイヤーも苦しめられる（魔神化で解除できる価値の担保） |
| 希少性 | waist残滓「状態異常命中」はメイン確率2.5%の激レア枠（doc11）。意図的に積みにくい設計 |

## 2. 状態異常の種別一覧

6種の状態異常を定義する。エネミーとプレイヤーの双方に付与できる。

| ID | 名称 | 主な属性 / 攻撃種別 | 効果 | 持続 | スタック |
|---|---|---|---|---|---|
| `BLEED` | 出血 | SLASH / STRIKE（物理） | 攻撃側ATK × 5% の固定ダメージ/ターン | 3ターン | **最大3** |
| `POISON` | 毒 | DARK / MAGIC | 対象maxHP × 3% のダメージ/ターン | 3ターン | 1（延長のみ） |
| `BURN` | 燃焼 | FIRE 属性 | 対象maxHP × 5% のダメージ/ターン | 2ターン | 1（上書き） |
| `FREEZE` | 凍結 | ICE 属性 | 1ターン行動不能（ターン終了で自動解除） | 1ターン | 1（上書き） |
| `PARALYSIS` | 麻痺 | THUNDER 属性 | ターン開始時50%の確率で行動スキップ | 2ターン | 1（延長のみ） |
| `WEAKEN` | 衰弱 | DARK / SUMMON | ATK -30%（乗算補正として適用） | 2ターン | 1（上書き） |

> **スタック仕様の定義**:
> - **最大3スタック**: 同一ターン内に複数回命中しても3を上限とする。各スタックの残りターンは独立して管理する。
> - **1（延長のみ）**: 付与済みの場合、残りターンをbaseDebuffDuration にリセットする。
> - **1（上書き）**: 常に新しい付与で残りターンをリセットする。

## 3. 発動判定ロジック

### 3.1 基本計算式（doc09 §8 の再掲）

```typescript
// src/logic/StatusAilmentSystem.ts

function calcDebuffChance(
  baseRate: number,       // 各状態異常のbaseDebuffRate（下記テーブル参照）
  attackerEffectHit: number,  // 攻撃側の effectHit（%）
  targetEffectRes: number,    // 対象の effectRes（%）
): number {
  const raw = baseRate * (1 + attackerEffectHit / 100) * (1 - targetEffectRes / 100);
  return Math.max(0, Math.min(1, raw));  // 0〜1 にクランプ
}

function tryApplyAilment(
  ailmentId: AilmentType,
  attacker: { effectHit: number; atk: number },
  target: { effectRes: number; statusEffects: StatusEffect[] },
): boolean {
  const baseRate = BASE_DEBUFF_RATES[ailmentId];
  const chance = calcDebuffChance(baseRate, attacker.effectHit, target.effectRes);
  if (Math.random() >= chance) return false;

  applyStatusEffect(target, ailmentId, attacker.atk);
  return true;
}
```

### 3.2 各状態異常の baseDebuffRate

| 状態異常 | baseDebuffRate | 設計意図 |
|---|---|---|
| BLEED | 0.35 (35%) | 物理攻撃の補助として発動しやすく、スタックで大きく育つ |
| POISON | 0.30 (30%) | 長期型DoTなので発動率はやや控えめ |
| BURN | 0.25 (25%) | 即効性が高い分、発動しにくい |
| FREEZE | 0.20 (20%) | 行動不能という強力CCのため最低発動率 |
| PARALYSIS | 0.25 (25%) | 確率的行動不能なのでFREEZEより少し高め |
| WEAKEN | 0.30 (30%) | ATKデバフは汎用性が高いため中程度 |

### 3.3 effectHit / effectRes の実用数値帯

| 数値帯 | 意味 | 到達手段 |
|---|---|---|
| effectHit 0% | 初期値（プレイヤー） | 素の状態 |
| effectHit 10〜20% | 状態異常補助ビルドの入口 | waist残滓「状態異常命中」+ 武器サブ |
| effectHit 30〜40% | 状態異常特化ビルドの完成形 | 複数部位への集中投資 |
| effectRes 0% | 無防備な雑魚モンスター | MINION の標準 |
| effectRes 5% | プレイヤー初期値 | 素の状態 |
| effectRes 10〜20% | 耐性装備の効果が出始める | legs残滓「状態異常耐性」積み |
| effectRes 30〜50% | ボス / ELITE の標準 | master データで設定 |

**計算例（effectHit=20%, target.effectRes=30% の場合）**:
```
POISON (35%) の実際の発動確率
= 0.30 × (1 + 20/100) × (1 - 30/100)
= 0.30 × 1.20 × 0.70
= 0.252 (25.2%)
```

## 4. 各状態異常の詳細仕様

### 4.1 出血（BLEED）

```
ターン開始時処理:
  damage = attacker.atk × 0.05 × stackCount
  target.currentHp -= damage
  stackCount > 0 → remainingTurns-- (各スタック独立)
  remainingTurns == 0 のスタックは除去
```

- 追加スタックは「攻撃者のATK」を記録して発動したターン時点で固定
- 3スタック × ターン3継続 = 最大9回のDoT攻撃（デバッフコンボの核）
- 魔神化での全解除対象（doc16 §2.2 と一致）

### 4.2 毒（POISON）

```
ターン開始時処理:
  damage = Math.floor(target.maxHp × 0.03)
  target.currentHp -= damage
  remainingTurns--
```

- 付与済みの場合: remainingTurns を 3 にリセット（ダメージ累積なし）
- 回復量の少ないエネミーへの持続プレッシャーに最適
- エリアギミック `STATUS_AILMENT`（エリア2・瘴気の沼）はこの毒を自動付与する

### 4.3 燃焼（BURN）

```
ターン開始時処理:
  damage = Math.floor(target.maxHp × 0.05)
  target.currentHp -= damage
  remainingTurns--
```

- DoT威力はPOISONより高い（5% vs 3%）が、持続が2ターンに限定
- 上書き方式のため再付与で延命できない → 「もう1発入れれば2ターンに戻る」テンポ管理が発生

### 4.4 凍結（FREEZE）

```
ターン開始時処理:
  target.isSkippingAction = true
  remainingTurns--  // remainingTurns が 0 になったら解除
```

- 強力なCC（確定行動不能）のため持続1ターンに厳しく制限
- FREEZEで凍結中のエネミーへの攻撃は WEAKNESS 表記を出す（演出強化）
- ボスへの発動確率は effectRes=50% 基準で: 0.20 × 1.0 × 0.50 = 10% → 十分レア

### 4.5 麻痺（PARALYSIS）

```
ターン開始時処理:
  if Math.random() < 0.50:
    target.isSkippingAction = true
  remainingTurns--
```

- FREEZEと違い毎ターン50%の確率判定 → 「解けるかもしれない」緊張感がある
- 持続2ターンで合計2回判定。期待値で1ターン行動不能と等価

### 4.6 衰弱（WEAKEN）

```
ダメージ計算時の処理:
  effectiveAtk = target.stats.atk × (isDemonized ? 1.0 : 0.70)
  // ← 魔神化中は無効化される（doc16 §2.2 の免疫と一致）
ターン終了時:
  remainingTurns--
```

- ATKを乗算で30%削減（additive ではなく multiplicative）
- 敵が使う場合もあり、プレイヤーへの圧力ギミックとして機能する

## 5. ボスギミック「タイムライン強制遅延」の数値仕様

doc15 §3 で定義された `BossGimmick { effect: 'AV_DELAY', value?: number }` の `value` と減衰式を確定する。

### 5.1 AV_DELAY の基本仕様

```
baseAVDelay = 40  // 行動値に加算するポイント

actualDelay = Math.floor(baseAVDelay × (1 - target.effectRes / 100))
target.currentAV += actualDelay
```

| プレイヤー effectRes | 実際の遅延 AV | 体感 |
|---|---|---|
| 0% | +40 AV | 大幅に後回しにされる（SPD=110基準でほぼ半ターン分） |
| 5% (初期値) | +38 AV | ほぼ同等の被害 |
| 20% | +32 AV | 若干の軽減 |
| 50% | +20 AV | 半減：legs+waist残滓を状態異常耐性で固めた場合の到達値 |
| 100% | 0 AV | 完全無効（装備限界を超えるため現実的でない） |

> **設計意図**: effectRes=50% で遅延を半減できる達成感を用意しつつ、
> 完全無効化を現実的に不可能にすることでボスギミックとしての脅威を維持する。

### 5.2 AV_DELAY トリガー条件

```typescript
// BossGimmick の適用例（enemies.json での記述）
{
  "trigger": "TURN_3",       // 3ターン目に発動
  "effect": "AV_DELAY",
  "value": 40                // baseAVDelay
}
```

- trigger は既存 `BossGimmick.trigger` の型（`HP_BELOW_50 | TURN_3 | ON_SHIELD_BREAK`）を使用
- `TURN_3` 固定でよいが、BOSSごとに `value` で強度を調整可能

## 6. BattleEngine へのステート管理設計

### 6.1 型定義への追加（`src/types/game.ts`）

```typescript
type AilmentType = 'BLEED' | 'POISON' | 'BURN' | 'FREEZE' | 'PARALYSIS' | 'WEAKEN';

interface StatusEffect {
  type: AilmentType;
  remainingTurns: number;
  stackCount: number;      // BLEED のみ > 1 になりうる、それ以外は常に 1
  sourceAtk?: number;      // BLEED の damage 計算用（付与時の攻撃側 ATK を固定記録）
}

// CharacterData に追加
interface CharacterData {
  // ... 既存フィールド ...
  statusEffects: StatusEffect[];   // 現在かかっている状態異常のリスト
}
```

### 6.2 BattleEngine 内の処理フロー

```typescript
// src/logic/BattleEngine.ts — processStatusEffects()
private processStatusEffects(target: CharacterData | EnemyRuntimeState): BattleLog[] {
  const logs: BattleLog[] = [];

  for (const effect of target.statusEffects) {
    switch (effect.type) {
      case 'BLEED':
        const bleedDmg = (effect.sourceAtk ?? 0) * 0.05 * effect.stackCount;
        target.currentHp -= Math.floor(bleedDmg);
        logs.push(this.makeAilmentLog(target, 'BLEED', bleedDmg));
        break;
      case 'POISON':
        const poisonDmg = Math.floor(target.stats.hp * 0.03);
        target.currentHp -= poisonDmg;
        logs.push(this.makeAilmentLog(target, 'POISON', poisonDmg));
        break;
      case 'BURN':
        const burnDmg = Math.floor(target.stats.hp * 0.05);
        target.currentHp -= burnDmg;
        logs.push(this.makeAilmentLog(target, 'BURN', burnDmg));
        break;
      case 'FREEZE':
        target.isSkippingAction = true;
        break;
      case 'PARALYSIS':
        if (Math.random() < 0.5) target.isSkippingAction = true;
        break;
      case 'WEAKEN':
        // ダメージ計算時に参照する（ここでは何もしない）
        break;
    }
    effect.remainingTurns--;
  }

  // 期限切れエフェクトを除去
  target.statusEffects = target.statusEffects.filter(e => e.remainingTurns > 0);
  return logs;
}
```

### 6.3 魔神化との接続（doc16 §2.2 と矛盾なし）

## 7. エリアギミックとの連動（doc03 §5 の補完）

doc03 §5 の `STATUS_AILMENT` エリアギミックは、ターン開始時に自動的に毒を付与する。

```typescript
// BattleEngine.ts — processAreaGimmick()
case 'STATUS_AILMENT':
  // 瘴気の沼エリア: 毎ターン開始時に 100% でプレイヤーへ POISON を付与
  // 通常の発動判定式は使わず、エリアギミックは確定付与とする
  applyStatusEffect(player, 'POISON', 0 /* sourceAtk 不要 */);
  break;
```

> **設計意図**: エリアギミックは「逃げられないプレッシャー」として機能するため、
> effectRes による軽減対象外とし、確定付与にする。

## 9. UI 実装ガイドライン

### 9.1 状態異常アイコン（エネミー頭上）

| 状態異常 | アイコン色 | 表示内容 |
|---|---|---|
| BLEED | 赤（#FF2244） | 滴のアイコン + スタック数バッジ |
| POISON | 紫（#9900FF） | ドクロのアイコン |
| BURN | オレンジ（#FF6600） | 炎のアイコン |
| FREEZE | 水色（#00CCFF） | 結晶のアイコン |
| PARALYSIS | 黄（#FFEE00） | 稲妻のアイコン |
| WEAKEN | グレー（#888888） | 下矢印アイコン |

### 9.2 演出ルール

- DoT 発動ターンはダメージ数字の色を通常白から**緑（ピュアダメージと区別）**に変更
- FREEZE 解除時: 「砕け散る氷」の短いパーティクル演出（1フレーム程度）
- 魔神化発動による一括解除: アイコンが閃光と共に全消え（カタルシス演出）
- WEAKEN 中の敵: 体色を10%グレー寄りに傾けてプレイヤーに視覚的フィードバック

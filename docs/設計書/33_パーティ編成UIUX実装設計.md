# 33. パーティ編成UI/UX実装設計

> 最終更新: 2026-05-16  
> 対象: `src/components/legion/LegionHub.tsx`

## 目的

LegionHub の編成体験を iPhone 13 Pro 相当のブラウザで扱いやすくする。常時表示の小さいロスターではなく、編成スロットを起点に「選ぶ」「比較する」「詳細を見る」「配置する」を段階化し、誤タップと情報不足を減らす。

## 画面遷移

```
編成トップ
  ├─ 編成スロットタップ / 魔物選択ボタン
  │    └─ 魔物選択ページ
  │          ├─ 魔物カードタップ: そのスロットへ配置
  │          ├─ 詳細: 魔物詳細ページ
  │          └─ ×: 編成トップへ戻る
  └─ DETAIL: ユニット詳細へ戻る

魔物詳細ページ
  ├─ 編成する: 対象スロットへ配置して編成トップへ戻る
  └─ ×: 魔物選択ページへ戻る
```

## 編成トップ

- 主役は主人公カード + 3つの編成スロット。
- ロスター/ソートは常時表示しない。
- 選択中スロットには「前衛/中衛/後衛」「ヘイト比率」「現在配置中の魔物名」を表示する。
- 編成スロットをタップすると、そのスロットを対象に魔物選択ページへ遷移する。編成トップの上にモーダルを重ねず、画面全体をページとして差し替える。
- 長押しドラッグによるスロット入れ替えは維持する。

## 魔物選択ページ

- ポップアップではなく、LegionHub 内のフルスクリーンページとして表示する。
- iOS 実機で左寄りにならないよう、`fixed` オーバーレイや中央シェルは使わず、LegionHub ルート配下の通常ページとして `relative flex h-full min-h-0 w-full flex-col` で描画する。
- コンテンツは `width: 100%; max-width: 430px; margin: 0 auto` を前提にし、iPhone 幅では全幅、広い検証環境では中央に来るようにする。カードやフッターはスマホ幅で押しやすい密度になるよう余白を確保する。
- Header:
  - `×` は編成トップへ戻る。
  - 対象スロット、現在の配置魔物、現在コスト/最大コストを表示。
- Filter:
  - 種族チップは 48px 四方。
  - CLEAR と Sort ボタンは 44px 以上。
- Monster Card:
  - COST を ATK/HP/SPD/DEF と同じステータスグリッドに表示する。
  - 既に編成済みの場合は VANGUARD/MIDDLE/REAR バッジを表示する。
  - コスト超過候補は赤系の境界線と COST 表示で警告する。
  - `詳細` ボタンから魔物詳細ページへ遷移する。

## 魔物詳細ページ

- 魔物選択ページと同じく画面全体を占有するページとして描画し、詳細内容だけが左寄せの小型モーダルに見えないようにする。
- `×` は魔物選択ページへ戻る。
- 表示項目:
  - 名前、種族、COST、対象スロット
  - COST/ATK/HP/SPD/DEF/CR/CD/E.HIT/E.RES
  - 属性耐性、弱点
  - Spirit Core がある場合は霊核情報
- 下部に `編成する` ボタンを配置する。
- コスト超過時は `編成する` を無効化し、現在コストと最大コストを赤で示す。
- 主要ステータスは2列グリッドで表示し、数値は 20px 前後まで大きくして片手操作中でも読める密度にする。

## iOS Safari ルール

- 画面の `motion.div` と `overflow:hidden` は分離する。
- 選択/詳細ページは `fixed` と `absolute inset-0` を避け、`LegionListView` の通常分岐として `relative flex h-full min-h-0 w-full flex-col` で表示する。iOS Safari で transformed parent が `fixed` の基準を変える問題や、親の高さ解決に失敗して下部が見切れる問題を避ける。
- Vite 実行時は `#root` がアプリルートになるため、`#__next` と同じく `width/height: 100%` を指定する。
- スクロールが必要な本文領域は `flex-1 min-h-0 overflow-y-auto safe-scroll` を使い、フッターは `shrink-0` として常に画面内に残す。
- 主要タップ領域は 44px 以上。
- `100vh` ではなく `100dvh` / `env(safe-area-inset-*)` を使用する。
- 小さな横スクロールUIはページ下部へ常駐させない。

## 実装メモ

- Zustand の `party` は常に3スロット固定。
- コスト判定は `necroStatus.maxCost` を参照する。
- `assignMonster(slotIndex, monster)` は成功時 `true`、コスト超過時 `false` を返す。
- 詳細ページは DB 永続化を行わず、`inventoryMonsters` の `MonsterData` を表示する。

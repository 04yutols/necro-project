## 1. 設計概要

戦闘パーティは **アルド（固定・プレイヤー操作）+ モンスター3体（コスト制・順番設定可）** で構成される。モンスターは **slot 0 / 1 / 2 の順に先頭〜後衛** を表し、先頭ほど敵のヘイトを集めやすい。

この「隊列」システムにより、ORC系モンスターを先頭に置いてタンクとして機能させたり、BEAST系を後衛に置いてスピードシナジーを維持しながら守る戦略性が生まれる。

## 2. 隊列（フォーメーション）とヘイトシステム

### 2.1 ポジション定義

| slot | ポジション名 | 日本語表示 | ヘイト重み |
|---|---|---|---|
| 0 | `VANGUARD` | 前衛 | **50%** |
| 1 | `MIDDLE` | 中衛 | **30%** |
| 2 | `REAR` | 後衛 | **20%** |

ヘイト重みは**敵の単体攻撃がどのターゲットに向かうか**の確率分布（モンスター3体合計 100%）。

### 2.2 ヘイトの適用ルール

```
敵の単体攻撃 → ターゲット決定
  [1] 生存しているモンスターのみ対象とする
  [2] 各生存モンスターのヘイト重みで重み付き抽選
  [3] 当選モンスターが敵の攻撃を受ける（モンスターの DEF で軽減）
  [4] モンスターが全滅している場合 → アルドが直接受ける
```

敵の全体攻撃（AOE）は全生存モンスター + アルド全員に均等ダメージ。

### 2.3 モンスターの HP と被ダメージ

モンスターは自身の `stats.hp` を持つ（バトル開始時に `currentHp = stats.hp` で初期化）。  
被ダメージ計算: `finalDmg = baseDmg × (1 - monster.def / (monster.def + 200))`

- モンスターの `currentHp ≤ 0` → そのモンスターは **撃破** 扱い（バトル中はパーティから除外）
- 撃破されたモンスターはバトル終了後に HP を全回復して復帰（永続消滅なし）

### 2.4 BattleEngine への組み込み

```typescript
// BattleEngine.ts — 既存 processMonsterActions() の前処理として追加

private selectEnemyTarget(aliveMonsters: MonsterData[]): MonsterData | null {
  if (aliveMonsters.length === 0) return null;
  const HATE_WEIGHTS = [50, 30, 20];
  const totalWeight = aliveMonsters.reduce((sum, m) => {
    const pos = this.monsters.indexOf(m);
    return sum + (HATE_WEIGHTS[pos] ?? 20);
  }, 0);
  let rand = Math.random() * totalWeight;
  for (const m of aliveMonsters) {
    const pos = this.monsters.indexOf(m);
    rand -= HATE_WEIGHTS[pos] ?? 20;
    if (rand <= 0) return m;
  }
  return aliveMonsters[aliveMonsters.length - 1];
}
```

## 3. 隊列の並び替えUI — 原神 / Star Rail レベル仕様

### 3.1 インタラクションモデル

**ロングプレス → ドラッグ → スワップ** 方式（Star Rail の隊列編成と同様）。

```
[長押し 400ms]
  → カードが浮き上がる (scale 1.06, shadow 増幅)
  → ドラッグ可能状態へ
[ドラッグ]
  → 他カードがドロップ目標として光る
  → ドラッグ中カードは半透明 (opacity 0.85)
[ドロップ]
  → 2カードの位置がスワップ (Framer Motion layout animation)
  → バイブ [15, 10] + `se_tap`
  → SynergyBanner がリアルタイムで更新
```

### 3.2 PortraitCard ドラッグ実装方針

`LegionHub.tsx` の `PortraitCard` に Framer Motion `drag` を追加する。  
**iOS Safari 制約遵守**: `motion.div` (drag/transform 担当) と その子の `div` (overflow:hidden 担当) を必ず分離する。

## 4. コスト超過 UI — 原神レベル仕様

### 4.1 コストゲージの段階表示

現在の実装（`LegionHub.tsx` L2452）は基本的な紫バーのみ。以下の3段階に拡張する:

| 状態 | 条件 | ゲージカラー | テキストカラー | 追加演出 |
|---|---|---|---|---|
| **余裕** | totalCost < maxCost × 0.75 | `#8B00FF → #DD22FF` (Void Purple) | `#E090FF` | — |
| **警戒** | totalCost ≥ maxCost × 0.75 | `#D4AF37 → #FFD700` (Cursed Gold) | `#FFD700` | ゲージが微細に脈動 |
| **超過** | totalCost > maxCost | `#FF4444 → #FF8800` (赤橙) | `#FF6666` | ゲージが振動 + `se_error` |

### 4.2 コスト超過時のフィードバック

```
モンスターをパーティに追加しようとしてコスト超過になる瞬間:
  [1] 追加を拒否（スロットに入らない）
  [2] コスト表示が赤に変わり 0.4s シェイク (x: ±4px × 3回)
  [3] 超過している各モンスターカードの cost タグが赤くハイライト
  [4] se_error (低音ハム) + バイブ [10, 5, 10]
  [5] 「コスト超過: {totalCost}/{maxCost}」のツールチップが 2s 表示
```

### 4.3 コスト表示コンポーネント仕様

```typescript
// LegionHub.tsx — CostIndicator コンポーネント（既存コスト表示を置き換え）

function CostIndicator({ totalCost, maxCost }: { totalCost: number; maxCost: number }) {
  const ratio = totalCost / maxCost;
  const isOver = totalCost > maxCost;
  const isWarn = ratio >= 0.75;

  const barColor = isOver
    ? 'linear-gradient(90deg, #FF4444, #FF8800)'
    : isWarn
      ? 'linear-gradient(90deg, #D4AF37, #FFD700)'
      : 'linear-gradient(90deg, #8B00FF, #DD22FF)';

  const textColor = isOver ? '#FF6666' : isWarn ? '#FFD700' : '#E090FF';

  return (
    <div>
      {/* テキスト */}
      <motion.span
        animate={isOver ? { x: [0, -4, 4, -4, 4, 0] } : {}}
        transition={{ duration: 0.4 }}
        style={{ color: textColor, fontFamily: 'monospace', fontSize: 12, fontWeight: 900 }}
      >
        COST {totalCost}/{maxCost}
        {isOver && <span style={{ fontSize: 9, marginLeft: 4 }}>OVER!</span>}
      </motion.span>

      {/* ゲージバー */}
      <div style={{ width: 100, height: 5, borderRadius: 999, background: 'rgba(0,0,0,0.42)', border: '1px solid rgba(136,0,228,0.2)', marginTop: 4, overflow: 'hidden' }}>
        <motion.div
          animate={isWarn && !isOver
            ? { opacity: [1, 0.6, 1] }  // 脈動
            : isOver
              ? { x: [0, -2, 2, -2, 0] }  // 振動
              : {}}
          transition={{ duration: isOver ? 0.3 : 1.5, repeat: Infinity }}
          style={{
            width: `${Math.min(100, ratio * 100)}%`,
            height: '100%',
            borderRadius: 999,
            background: barColor,
            boxShadow: isOver ? '0 0 6px #FF4444' : isWarn ? '0 0 6px #D4AF37' : 'none',
          }}
        />
      </div>

      {/* 超過ラベル（AnimatePresence で出現） */}
      <AnimatePresence>
        {isOver && (
          <motion.div
            initial={{ opacity: 0, y: -4 }} animate={{ opacity: 1, y: 0 }} exit={{ opacity: 0 }}
            style={{ fontSize: 9, color: '#FF6666', fontFamily: 'monospace', marginTop: 2 }}
          >
            ▲ コスト超過 — モンスターを外してください
          </motion.div>
        )}
      </AnimatePresence>
    </div>
  );
}
```

type SortKey = 'atk' | 'hp' | 'spd' | 'cost';
type SortDir = 'asc' | 'desc';

const [sortKey, setSortKey] = useState<SortKey>('atk');
const [sortDir, setSortDir] = useState<SortDir>('desc');

const handleSortPress = (key: SortKey) => {
  if (sortKey === key) {
    setSortDir(d => d === 'desc' ? 'asc' : 'desc');
  } else {
    setSortKey(key);
    setSortDir(key === 'cost' ? 'asc' : 'desc');
  }
};

const sortedMonsters = useMemo(() => {
  const filtered = inventoryMonsters.filter(m =>
    activeFilters.size === 0 || activeFilters.has('ALL') || activeFilters.has(m.tribe)
  );
  return filtered.sort((a, b) => {
    const va = sortKey === 'cost' ? a.cost : a.stats[sortKey];
    const vb = sortKey === 'cost' ? b.cost : b.stats[sortKey];
    return sortDir === 'desc' ? vb - va : va - vb;
  });
}, [inventoryMonsters, activeFilters, sortKey, sortDir]);

```typescript
swapPartySlots: (indexA, indexB) =>
  set(s => {
    const next = [...s.party] as (MonsterData | null)[];
    [next[indexA], next[indexB]] = [next[indexB], next[indexA]];
    return { party: next };
  }),

setParty: (party) => set({ party }),

damageMonster: (id, dmg) =>
  set(s => ({
    monsterCurrentHp: {
      ...s.monsterCurrentHp,
      [id]: Math.max(0, (s.monsterCurrentHp[id] ?? 0) - dmg),
    },
  })),

resetMonsterHp: () =>
  set(s => ({
    monsterCurrentHp: Object.fromEntries(
      (s.party.filter(Boolean) as MonsterData[]).map(m => [m.id, m.stats.hp])
    ),
  })),
```

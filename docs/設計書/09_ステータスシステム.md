# 09 — ステータスシステム（HSR互換設計）

> 最終更新: 2026-05-06  
> 実装ファイル: `src/types/game.ts`, `src/logic/BattleEngine.ts`, `prisma/schema.prisma`

## 1. 設計思想

崩壊スターレイル（HSR）のステータス体系を参考に、旧9統計制から刷新。

**旧システムの問題点:**
- 物理（ATK/DEF）と魔法（MATK/MDEF）の二元論 → 職業選択がステータス面で強制的
- LUCK（会心率）と TEC（会心ダメージ）が役割が重複しわかりにくい
- AGI は実際のターン順序に未接続だった

**新システムのコンセプト:**
- **単一 ATK** — 物理・魔法の区別はスキルの属性ダメージ加成で表現
- **会心率 + 会心ダメージ** を独立した数値として公開（HSR と同一インターフェース）
- **速度（SPD）** がターン順序を直接決定
- **エネルギー** が MP を置換（奥義発動リソース）
- **効果命中 / 効果抵抗** で将来の状態異常システムに備える

| 旧 | 新 | 備考 |
|---|---|---|
| `hp` | `hp` | 据え置き |

| `atk` | `atk` | 物理・魔法共通の攻撃力 |

| *(新規)* | `effectHit` | 効果命中 % |
| *(新規)* | `effectRes` | 効果抵抗 % |

## 3. BaseStats 完全定義

```typescript
// src/types/game.ts
interface BaseStats {
  hp:        number;  // 最大HP            (プレイヤー初期: 7200)

  effectRes: number;  // 効果抵抗 %        (プレイヤー初期: 5)
}
```

## 4. 新ダメージ計算式（HSRスタイル）

```typescript
// src/logic/BattleEngine.ts — calculateDamage()

// 1. 基礎ダメージ（スキル威力倍率を乗算）
baseDmg = ATK × skillPower

// 2. 防御軽減（HSR方式を簡易化）
//    DEF=720 → 720/(720+200) = 0.783 → 21.7%軽減
//    DEF=0   → 軽減なし（defMult = 1.0）
defMult = 1 - DEF / (DEF + 200)
afterDef = baseDmg × defMult

// 3. 属性ダメージ加成（装備・残滓のサブオプションから）
elementBoost = attacker.elementDmgBoosts[element] / 100  // 例: 10 → +10%
afterElement = afterDef × (1 + elementBoost)

// 4. 属性耐性（敵の抵抗値）
resMult = 1 - (enemyResistances[element] ?? 0) / 100
isWeakness = (enemyResistances[element] ?? 0) < 0   // 弱点: resistance < 0
isResisted  = (enemyResistances[element] ?? 0) > 0   // 耐性: resistance > 0
afterRes = afterElement × resMult

// 5. 会心判定
isCrit = Math.random() * 100 < critRate   // critRate=8 → 8%の確率
if (isCrit): afterRes × (critDmg / 100)  // critDmg=165 → ×1.65

// 6. 最低保証
finalDmg = Math.max(1, Math.floor(afterRes))
```

### 旧式との比較

| 指標 | 旧式 | 新式（HSR） |
|---|---|---|
| 基礎計算 | `ATK² / (ATK + DEF)` | `ATK × power × (1 - DEF/(DEF+200))` |
| 防御の効き方 | DEFが高いほど急激に軽減 | DEFが高いほど緩やかに軽減（上限なし） |
| 会心率 | LUCK % | critRate % |
| 会心ダメージ | `1.5 + TEC/200` | `critDmg / 100`（固定倍率） |
| 属性ダメージ加成 | なし（耐性のみ） | elementDmgBoost で補正可能 |

## 5. エネルギーシステム

MP を廃止し、エネルギーゲージに置換。奥義発動リソースとして機能する。

| トリガー | エネルギー変化 |
|---|---|
| 通常攻撃実行 | +20 |
| 魔法スキル（通常）実行 | +15 |
| 敵から攻撃を受ける | +10 |
| 魔神化発動 | -30 |
| 奥義（Ultimate）発動 | -maxEnergy（全消費） |

`maxEnergy` はジョブ定義の `energyCurve.baseMaxEnergy` から決まる。  
エネルギーは `CharacterData.currentEnergy` としてランタイムに管理し、DB には保存しない。

### ジョブ別 maxEnergy

| ジョブ | maxEnergy | 特徴 |
|---|---|---|
| warrior | 100 | 標準 |
| mage | 80 | 奥義が軽い → 頻度高 |
| dark_priest | 80 | 同上 |
| rogue | 90 | やや軽め |
| dark_knight | 110 | 重い → 威力高 |
| necromancer | 120 | 最重 → 最強奥義 |
| berserker | 120 | 重い連撃型 |
| archmage | 100 | 広範囲魔法 |
| assassin | 90 | 素早い奥義 |

## 6. 速度（SPD）と行動順序

HSR の Action Value 方式を採用（簡易実装版）。

```
行動値（AV）= 10000 / SPD
```

- **AV が小さいほど先に行動**
- SPD=110 → AV≈91, SPD=80 → AV=125
- 現在実装: ターン開始時に `player.spd > enemy.spd` で先攻判定
- 将来拡張: 全エンティティのAVキューを管理し、最小AV順に行動

### SPD 参考値

| エンティティ | SPD | 位置付け |
|---|---|---|
| 主人公（初期） | 110 | 標準より少し速め |
| warrior 修正後 | 108 | 遅め前衛 |
| rogue 修正後 | 130 | 高速 |
| goblin | 80 | 標準的な雑魚 |
| boss_goblin | 100 | 標準 |
| zombie | 20 | 超低速 |

## 8. 効果命中・効果抵抗

将来実装する状態異常システムへの準備。

```
debuff付与確率 = baseDebuffRate × (1 + effectHit/100) × (1 - target.effectRes/100)
```

- effectHit=20, target.effectRes=10 → 1.2 × 0.9 = 1.08倍
- 現在は BattleEngine に未接続（状態異常実装時に追加）

## 9. 属性ダメージ加成（elementDmgBoost）

装備・残滓のサブオプションから加算される。`CharacterData` に派生フィールドとして保持。

```typescript
// CharacterData に追加
elementDmgBoosts: Partial<Record<ElementType, number>>;
// 例: { FIRE: 15, DARK: 10 } → 炎+15%、闇+10%
```

装備変更時に `useGameStore` 側で再計算し更新する。

```typescript
// SubOption タイプ例
{ type: "FIRE_DMG_BOOST",   value: 10 }   // 炎ダメージ +10%
{ type: "DARK_DMG_BOOST",   value: 8  }   // 闇ダメージ +8%
{ type: "VOID_DMG_BOOST",   value: 12 }   // 全属性 +12%（特殊）
```

### プレイヤー初期値

| ステータス | 値 | 備考 |

| HP | 7200 | 序盤は十分な耐久 |
| ATK | 1250 | 旧 atk=1250 を継承 |
| DEF | 720 | 旧 def=720 を継承（mdefは吸収済み） |
| SPD | 110 | 平均的なモンスターより少し速め |
| CRIT RATE | 8% | 基礎5% + job修正 |
| CRIT DMG | 165% | 基礎150% + 少し盛り気味 |
| EFFECT HIT | 0% | 初期は0（装備で積む） |
| EFFECT RES | 5% | 少しの状態異常耐性 |

### モンスター初期値（例）

| モンスター | HP | ATK | DEF | SPD | critRate | critDmg |
|---|---|---|---|---|---|---|
| Goblin | 50 | 10 | 5 | 80 | 0 | 150 |
| Skeleton | 40 | 12 | 8 | 50 | 0 | 150 |
| Zombie | 80 | 8 | 4 | 20 | 0 | 150 |
| Imp | 60 | 15 | 10 | 120 | 5 | 155 |
| Goblin King | 300 | 40 | 30 | 100 | 10 | 150 |

### ダメージ期待値（warrior Lv1、対Goblin）

```
baseDmg = 1250 × 1.0 (通常攻撃) = 1250
defMult = 1 - 5/(5+200) = 0.976
finalDmg = 1250 × 0.976 = 1220
```

旧式での同じケース: `1250² / (1250+5) ≈ 1244` — ほぼ同等のレンジを維持。

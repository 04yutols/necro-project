本仕様は、スタミナ無制限の環境下でプレイヤーが何千回も周回することを前提とし、
**「アクセスへのストレスゼロ」「1周2〜3分の超高速テンポ」「明確な役割を持った敵の配置」**
を至上命題とするコアゲームループの設計である。

## 1. 空間の階層構造（ワールド〜ダンジョンの流れ）

プレイヤーがログインしてから剣を振るうまでの導線設計。
周回ゲーであるため、遷移は極力直感的かつ高速に行う。

```
ワールドマップ（Layer 1）
  └── エリアマップ（Layer 2）
        ├── 安全地帯（ネクロラボ / 拠点機能）
        └── 深淵の迷宮ノード × n（Layer 3 = バトル）
```

---

### Layer 1 — ワールドマップ（全体図）

世界全体を見渡す壮大なマップ。メインストーリーの進行（章）ごとに新しい大陸・領域が解放される。

| 章 | エリア名（仮） | 解放条件 |
|---|---|---|
| 第1章 | 亡国の王都 | ゲーム開始時から開放 |
| 第2章 | 腐海の大森林 | 第1章 全ノードクリア |
| 第3章 | 煉獄の深淵回廊 | 第2章 全ノードクリア |
| ... | ... | ... |

> ワールドマップは「どこへ向かうか」を選ぶ画面。詳細な探索はエリアマップで行う。

---

### Layer 2 — エリアマップ（探索と拠点）

ワールドマップから選択した章の詳細マップ。ノード（点）がツリー状に点在し、
未クリアのノードを攻略することで隣接ノードの霧が晴れて進行できる。

| ノード種別 | 表示 | 機能 |
|---|---|---|
| 安全地帯 | 拠点アイコン | ネクロラボ・転職・編成・分解が利用可能 |
| 深淵の迷宮 | ダンジョンアイコン | タップで即バトル突入（Layer 3へ） |
| 霧に閉ざされたノード | グレーアウト | 前提ノードクリアで解放 |
| ボスノード | 特殊アイコン | 章ボスが待ち受け、クリアで次章解放 |

**ツリー進行の設計意図**: プレイヤーに「次の目標」を常に明示しながら、
ハクスラの「同じダンジョンを何百回も周回する」行動を正当化する地図構造を提供する。

```
// 将来実装: src/data/master/stages.json 追加フィールド例
{
  "id": "area1_node3",
  "name": "煤けた王城の地下牢",
  "layer": 2,
  "waveCount": 3,
  "unlockRequires": ["area1_node1", "area1_node2"],
  "bossId": "elite_undead_knight",
  "rewards": { ... }
}
```

---

### Layer 3 — ダンジョン（バトルフェーズ）

エリアマップのノードを選択すると、**直接バトル専用ダンジョンに突入**。
無駄な移動・探索フェーズは挟まない。

## 2. ダンジョン自体の設計（無限周回特化フォーマット）

### 2.1 基本構造: 3WAVEバトル制

1周「**約2〜3分**」で完結し、ドロップリザルト画面へ直行できるテンポ感を基本とする。
複雑な迷路歩きは完全に排除し、3回の連続バトルで構成する。

```
ダンジョン入場
  │
  ├─ WAVE 1: 露払い（雑魚 × n）
  │     └─ 目的: 魔神化ゲージを溜め、バフ準備を整える
  │
  ├─ WAVE 2: 障壁（Elite + 雑魚）
  │     └─ 目的: 弱点属性「霊魂砕き」ギミックの突破
  │
  └─ WAVE 3: 死闘（Boss）
        └─ 目的: 全力で殴り倒してドロップリザルトへ
```

### 2.2 WAVEごとのゲームプレイ設計意図

| WAVE | 役割 | キーメカニクス | テンポ |
|---|---|---|---|
| WAVE 1 (露払い) | ウォームアップ＆ゲージ蓄積 | 魔神化ゲージ充填。スキルの気持ちよさを享受 | 速攻（30〜45秒） |
| WAVE 2 (障壁) | 編成・属性の適正テスト | 霊的防壁（属性シールド）の「霊魂砕き」が必須 | 中速（45〜60秒） |
| WAVE 3 (死闘) | クライマックス | 魔神化＋完凸武器パッシブの爆発。ボスギミック処理 | 緊張感（45〜60秒） |

### 2.3 オート周回と速度設定

完全スキップ（掃討チケット等）は**実装しない**。
「操作する楽しさ・眺める楽しさ」を損なわないことを優先する。

| 機能 | 仕様 |
|---|---|
| **フルオート戦闘** | AIが行動を自動選択。スキル・魔神化・霊魂砕きも自動判断 |
| **2倍速** | バトルエンジン（PixiJS）の描画フレームを2倍速で進行 |
| **3倍速** | 同上、3倍速。最速周回モード |
| 掃討チケット | **実装しない**（ハクスラの「ドロップ瞬間の体験」を守るため） |

> **設計根拠**: スキップは「ドロップ演出の感動」を消去する。
> フルオート＋3倍速で1周90秒前後を実現し、スキップなしでも周回疲弊を防ぐ。

## 3. 魔物（エネミー）の階層と役割定義

敵の強さ・ギミックをドロップするアイテムのレアリティと完全にリンクさせる。

---

### 階層① — 彷徨う骸（Minions / 雑魚）

| 項目 | 仕様 |
|---|---|
| HP | 低（プレイヤーの通常攻撃 1〜2発で撃破） |
| ギミック | なし |
| 行動 | 単調な近接攻撃のみ |
| **役割** | 一騎当千の爽快感を与える。**魔神化ゲージの餌** |
| **ドロップ** | 武器経験値素材、**R（凡庸）武器** |

> **設計意図**: WAVE 1 の雑魚はプレイヤーのウォームアップ兼ゲージ充填装置。
> この段階でストレスを与えず、次のWAVEへの期待を高めることが目的。

---

### 階層② — 深淵の眷属（Elites / 精鋭・中ボス）

| 項目 | 仕様 |
|---|---|
| HP | 中〜高（プレイヤーの数ターン分のダメージに耐える） |
| **コアギミック** | **霊的防壁（属性シールド）**: 弱点属性以外のダメージを最大80%カット |
| 霊魂砕き | 弱点属性で規定ダメージを与えることでシールドを破壊。破壊後は「撃破判定」でボーナス効果が発動 |
| 行動 | 高火力の属性攻撃。防壁展開中は無敵に近い |
| **役割** | 属性ビルドの適正テスト。**門番** |
| **ドロップ** | **SR（業物）武器**、中級の深淵の残滓 |

**霊的防壁（属性シールド）の詳細仕様**:

```
霊的防壁:
  弱点属性ダメージ以外: × 0.20（80%カット）
  弱点属性ダメージ:     × 1.00（通常計算）
  
霊魂砕き発動条件:
  弱点属性ダメージの累積 ≥ EliteHP × 0.3（30%相当）
  
霊魂砕き発動時の効果:
  - 防壁を完全解除（以降のカットなし）
  - 敵に「撃破状態（Break）」を付与: 次の行動まで行動不能 + 被ダメ +50%
  - プレイヤーのエネルギーを +30 付与（ご褒美）
```

> **設計意図**: 「弱点を突く」という行為に明確なゲームプレイ上の報酬（撃破ボーナス）を与えることで、
> パーティ編成（属性選択）を意味のある戦略判断として機能させる。

---

### 階層③ — 階層主・大魔神（Bosses / レイドボス）

| 項目 | 仕様 |
|---|---|
| HP | 非常に高い（プレイヤーの全力攻撃を10〜20回分耐える設計） |
| ビジュアル | 巨大。ステージ全体に存在感を示す画面演出 |
| **コアギミック** | 複数の特殊行動（後述） |
| **役割** | 強化限界を突破した武器と魔神化で叩き潰す**巨大サンドバッグ兼最終試験** |

**ボスの特殊行動一覧**:

| ギミック名 | 効果 | 対処法 |
|---|---|---|
| **大技（Enrage）** | HP50%以下で発動。次ターンに即死級のダメージ | 魔神化＋シールドスキル／回避行動 |
| **タイムライン強制遅延** | プレイヤーの行動値（AV）を強制的に増加させる | effectRes（効果抵抗）が高いと軽減 |
| **霊的防壁（重装甲版）** | Eliteより硬い防壁。複数属性への弱点が分散 | 複数の属性バフを持つ編成が有効 |
| **復活（フェーズ2移行）** | HP0時に一度だけ即座に復活し、全ギミックをリセット | 連続して高火力を叩き込む必要がある |

**ドロップ設計**:

| ドロップ対象 | 確率（目安） | 条件 |
|---|---|---|
| **SSR（遺物）武器** | 0.5〜2.0% | 通常ドロップ |
| 最高レアの深淵の残滓 | 5〜15% | 通常ドロップ |
| **ボスの記憶の欠片** | 20〜30% | SSR確定クラフト用の専用素材 |
| **UR（神話）武器** | **???（実質0.01%未満）** | 【理外の顕現】 後述 |

## 4. UR（神話）武器の発現システム

UR武器はドロップリストに記載されない「**理外の呪装**」。
通常のガチャ確率の概念が適用されない特別な存在として設計する。

### 発現のトリガー設計

```
発現条件:
  ボス討伐時、実質 0.01% 未満の超低確率で
  ドロップリストに禍々しい呪装（UR）が紛れ込む

保証なし:
  ・天井（確定枠）: 実装しない
  ・周回数カウント: 表示しない
  ・確定クラフト: 実装しない
  
完全な RNG のみに依存する「純粋な宝くじ枠」
```

### 世界観との接続

> プレイヤーがそのダンジョンを何百周と狂ったように周回し、
> 大量虐殺された魔物たちの怨念と絶望が特異点に達した時、
> 極稀に禍々しい武器として物理領域に顕現する。

- **ドロップリストへの非掲載**: プレイヤーは存在を知りながら確率を知ることができない
- **発現時の演出**: 通常のSSR演出とは完全に異なる禍々しいVFX（暗転・血飛沫・怨念のパーティクル）
- **コミュニティ効果**: 「引いた」報告がSNSで爆発的に拡散し、他プレイヤーの周回意欲を刺激する

## 5. ドロップ確率の全体設計（WAVEと敵階層の対応）

```
WAVE 1（雑魚）      ─── R武器（高頻度）, 経験値素材
WAVE 2（Elite）     ─── SR武器（中頻度）, 中級残滓
WAVE 3（Boss）      ─── SSR武器（低頻度）, 最高級残滓, 記憶の欠片, [UR極稀]
```

| アイテム種別 | 1周あたりドロップ期待数（目安） |
|---|---|
| R武器 | 1〜3個（ほぼ確定） |
| SR武器 | 0〜1個（約14.5%） |
| SSR武器 | 0〜1個（約0.5%) |
| 中級残滓 | 1個（WAVE 2クリア確定） |
| 最高級残滓 | 0〜1個（約30〜50%） |
| 記憶の欠片 | 0〜1個（約20〜30%） |
| UR武器 | 0〜1個（0.01%未満） |

> **ループの自己強化**: R/SRが大量に落ちるほどイデア（突破素材）が増え → 本命武器が強化され →
> より難しいダンジョンに挑め → より良いSSRや残滓がドロップする。

## 6. エネミーマスターデータの構造

```typescript
// 実装: src/data/master/enemies.json
interface EnemyData {
  id: string;
  name: string;
  nameJa: string;
  tier: 'MINION' | 'ELITE' | 'BOSS';
  tribe: Tribe;                           // 既存: UNDEAD | DEMON | BEAST | HUMANOID
  stats: BaseStats;                       // 既存の BaseStats 型を流用
  weaknesses: ElementType[];              // 弱点属性（霊魂砕きの対象）
  shieldHp?: number;                     // 霊的防壁HP（ELITE/BOSSのみ）
  gimmicks?: BossGimmick[];              // BOSSのみ
  dropTable: DropEntry[];
}

interface BossGimmick {
  trigger: 'HP_BELOW_50' | 'TURN_3' | 'ON_SHIELD_BREAK';
  effect: 'ENRAGE' | 'AV_DELAY' | 'REVIVE' | 'SUMMON_MINIONS';
  value?: number;                         // 遅延量・ダメージ倍率等
}

interface DropEntry {
  type: 'WEAPON' | 'RESIDUE' | 'MATERIAL';
  rarity?: string;
  rate: number;                           // 0.0〜1.0
  isHidden?: boolean;                     // true = URドロップリスト非掲載
}
```

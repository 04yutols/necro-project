# 12 — 深淵の残滓 UI/UX 実装設計

> 最終更新: 2026-05-06  
> 参照: [11_深淵の残滓システム.md](11_深淵の残滓システム.md)  
> 実装ファイル: `src/logic/ResidueScore.ts`, `src/components/legion/LegionHub.tsx`, `src/store/useGameStore.ts`

## 1. 目的

設計書11で更新された深淵の残滓を、単なる追加ステータス装備ではなく、原神/崩壊スターレイル級の「厳選・比較・強化・救済」を持つビルド体験として扱う。

今回のUIでは、以下を第一優先にする。

| 体験 | 実装方針 |
|---|---|
| 部位を理解できる | 5部位ロードアウトを常時表示し、選択部位で一覧を自動フィルタ |
| 良品か即判断できる | 残滓スコアと C/B/A/S/SS 評価を全カードに表示 |
| 装備判断が速い | 選択中の残滓と装備中スロットのスコア差分を表示 |
| 強化の期待/絶望を見せる | Lv4/8/12/16/20 の Tier 履歴レーンを表示 |
| ハズレ品にも価値を持たせる | 魂の錬成タブでポイント・部位別製造権・必要コストを表示 |

## 2. 5部位ロードアウト

`equippedResidueSlots` は以下の順序で固定する。

| index | slotId | 表示名 | 役割 |
|---|---|---|---|
| 0 | `head` | 思念の兜 | HP固定 |
| 1 | `arms` | 剛力の籠手 | ATK固定 |
| 2 | `chest` | 骸の胸当て | 汎用 |
| 3 | `waist` | 深淵の帯 | 属性特化 |
| 4 | `legs` | 霊獣の具足 | 会心特化 |

残滓データは `AbyssalResidueData.itemId` を部位IDとして扱う。  
UIと `useGameStore.equipResidueToSlot()` は、部位が一致しない装備を拒否する。

## 3. 残滓スコア

スコア式は設計書11の式を採用する。

```text
残滓スコア = critRate × 2 + critDmg + atk% × 1.5 + hp%
```

`src/logic/ResidueScore.ts` に集約し、カード・詳細パネル・強化画面で同じ値を使う。

| 評価 | スコア |
|---|---|
| C | 0〜19 |
| B | 20〜34 |
| A | 35〜49 |
| S | 50〜64 |
| SS | 65〜 |

## 4. 画面構成

### 装備タブ

- 上部: 5部位ロードアウト
- 中央: 選択残滓の詳細、スコア、主効果、サブ効果、装備中との差分
- 下部: 選択部位の残滓一覧。スコア順に並び、部位外の残滓は表示しない

### 強化タブ

- 上部: 5部位ロードアウト
- 選択部位の候補を横スクロール表示
- 強化前後の主効果プレビュー
- Tier履歴レーンを表示
- 素材一括選択と強化ボタンを維持

### 錬成タブ

- 現在の錬成ポイントを表示
- 部位別の製造権コストを表示
- 選択部位の製造条件を表示
- 今回はUI導線まで実装し、実際の製造トランザクションは `ResidueService` 実装時に接続する

## 5. iOS/レスポンシブ方針

- 画面全体は既存の `motion.div` + 内側 `overflow-hidden` 構造を維持する
- スロットレーンは5列固定だが、各ボタンを短いラベル・固定高さにして iPhone 幅で崩れないようにする
- 詳細パネルは `min-width: 0` と省略表示を徹底し、長い残滓名でレイアウトが押し出されないようにする
- 一覧は仮想グリッドを維持し、カード高さを固定してスクロール中のジャンプを防ぐ

## 6. 今後の接続予定

- `ResidueSystem.ts`: 排出確率と4層RNG
- `ResidueService.ts`: 強化時のサブステータス選出、Tier抽選、錬成ポイント消費
- `prisma/schema.prisma`: `residueScore`, `tierHistory`, `transmutationPoints`
- 強化演出: Tier4 成功時のゴールド粒子、Tier1 成功時の暗転演出

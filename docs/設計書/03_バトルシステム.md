## 1. WAVEバトル進行フロー

```
ターン開始
  ↓
[1] エリアギミック処理 (processAreaGimmick)
  ↓
[2] プレイヤー行動 (processPlayerAction)
     ├─ PHYSICAL_ATTACK: Energy消費0、ATKダメージ、Energy +20
     └─ MAGIC_SKILL: Energyを消費 → MasterDataService からスキルデータ取得 → 計算
  ↓
[3] 軍団の追撃 (processMonsterActions)
     └─ パーティ内モンスター全員が対象モンスターを攻撃（PHYSICAL / NONE属性）
        └─ player.isAwakened == true の場合: ATK を1.5倍で適用
  ↓
[4] ターン・WAVE更新 (updateState)
     └─ turn++ → turn > 10: wave++ (max 3), turn = 1 にリセット
  ↓
BattleLog[] を返す → BattleCanvas.tsx が受け取りPixiJS演出を再生
```

## 2. ダメージ計算式（最新版）

```
// ステータスは src/logic/StatSystem.ts の total を使用する
// 物理/魔法は攻撃種別とVFXの差であり、計算軸は ATK/DEF に統一する

baseDmg = ATK × skillPower

// 防御軽減（HSR簡易式）
defMult = 1 - DEF / (DEF + 200)

// 属性ダメージ加成（装備・深淵の残滓・魂片由来）
elementBoostMult = 1 + elementDmgBoost[element] / 100

// 属性耐性補正
resMult = 1 - resistance[element] / 100
  resistance = -50 → 1.5× (弱点 isWeakness=true)
  resistance = 0   → 1.0× (等倍)
  resistance = 50  → 0.5× (耐性 isResisted=true)
  resistance = 100 → 0.0× (無効)

// 会心判定
isCritical = Math.random() * 100 < critRate

// 会心ダメージ
critMult = critDmg / 100

// 最終ダメージ
finalDamage = baseDmg × defMult × elementBoostMult × resMult × (isCritical ? critMult : 1)

// 最低保証
finalDamage = Math.max(1, Math.floor(finalDamage))
```

### スキル威力（power）の目安

| 威力 | 用途例 |
|---|---|
| 1.0 | 通常攻撃 |
| 1.25〜1.5 | Tier1 物理スキル |
| 1.5〜1.8 | Tier1 魔法スキル |
| 2.0 | Tier2 強力スキル（例: skill_necromancer_1） |

## 3. 属性（9種）× 攻撃種別（6種）

### 属性定義

| 属性 | ID | 主な用途 |
|---|---|---|
| 炎 | `FIRE` | 火球、爆炎、継続ダメージ |
| 水 | `WATER` | 水槍、水流、渦 |
| 雷 | `THUNDER` | 落雷、感電、雷斬 |
| 土 | `EARTH` | 岩撃、地割れ |
| 風 | `WIND` | 風刃、広域斬撃 |
| 光 | `LIGHT` | 勇者/天界系、アンデッドに有効 |
| 闇 | `DARK` | 死霊術/魔王系 |
| 氷 | `ICE` | 上位派生・エリア限定 |
| 無 | `NONE` | 通常攻撃・無属性追撃 |

### 攻撃種別定義

| 種別 | ID | VFX基準 |
|---|---|---|
| 斬撃 | `SLASH` | 斜めの軌跡、刃筋、連続斬線 |
| 打撃 | `STRIKE` | 地面衝撃、波紋、重いヒット |
| 射出 | `PROJECTILE` | 射出体、着弾 |
| 術式 | `MAGIC` | 魔法陣、詠唱円、範囲展開 |
| 召喚 | `SUMMON` | 召喚陣、使役魔の影 |
| 回復 | `HEAL` | 上昇粒子、柔らかい光 |

### VFX解決フロー

```
1. skillData.effectKey が存在 → 専用VFX（最優先）
2. effectKey なし → element × attackType の共通VFX
3. element なし → NONE として処理
4. attackType なし → 魔法系は MAGIC / 物理系は SLASH にフォールバック
```

## 4. スキルデータ構造（SkillData）

```typescript
interface SkillData {
  id: string;
  name: string;
  mpCost: number;         // Energy消費量（旧フィールド名をマスターデータ互換で維持）
  power: number;          // ダメージ倍率
  type: 'PHYSICAL' | 'MAGICAL' | 'HEAL';  // スキル分類・UI/演出傾向
  element?: ElementType;  // 属性（耐性計算軸）
  attackType?: SkillAttackType;  // 演出軸
  targetType?: 'SINGLE' | 'ALL_ENEMIES' | 'SELF' | 'ALLY';
  effectKey?: string;     // 専用VFXキー（未登録なら共通VFX）
  description: string;
}
```

## 6. 魔神化システム

> **詳細設計は [16_魔神化システム.md](16_魔神化システム.md) を参照。**

### 実装仕様（2026-05-09 更新）

| 要素 | 詳細 |
|---|---|
| ゲージ | `demonGauge`: 0〜100 |
| 発動条件 | `demonGauge >= 100` かつ `isDemonMode == false` |
| 発動 | 任意発動。敵ターン中でも発動でき、敵行動タイマーをキャンセルして自ターンへ絶対割り込み |
| 継続 | 主人公が3回行動するまで（ターン数ではなく行動数基準） |
| ゲージ変化 | 通常行動: +10 / WAVEクリア: +18 / 発動時に0へ消費。魔神化中の毎ターン-20消費は廃止 |
| 効果A | 職業ごとの `effectA` を適用。旧一律+20%補正は廃止 |
| 効果B | 職業ごとのギミックとリスクを適用。例: 3回ヒット、紙装甲、自傷、過剰消費、仕込み依存 |
| 状態異常 | 魔神化中は状態異常・デバフを完全無効化する前提で扱う |
| 魔神技 | 魔神化中に1度だけ専用奥義を使用可能。使用後はボタンがグレーアウト |
| 解除 | 3回行動経過 または `toggleDemonMode()` / `endDemonMode()` |

## 7. BattleCanvas — BattleEngine 連携詳細

BattleCanvas（フロントエンド）は BattleEngine（純粋クラス）を直接インスタンス化せず、`useGameStore.actionTrigger` 経由で疎結合に接続する。

```typescript
// BattleCanvas.tsx 内
useEffect(() => {
  if (!actionTrigger || !battleEngine) return;

  const logs = battleEngine.simulateAction(
    actionTrigger.type,
    targetMonster,
    actionTrigger.skillId
  );

  // BattleLog の element + attackType でVFXを選択
  logs.forEach(log => {
    playVFX(log.element, log.attackType);
    if (log.isCritical) showCritText();
    if (log.isWeakness) showWeakText();
  });

  setActionTrigger(null);  // トリガーリセット
}, [actionTrigger]);
```

### BattleLog の主要フィールド

```typescript
interface BattleLog {
  turn: number; wave: number;
  action: string;
  actorName: string; targetName: string;
  damage?: number;
  isCritical?: boolean;
  isWeakness?: boolean;   // UI: "WEAK!" 表示
  isResisted?: boolean;   // UI: "RESIST" 表示
  element: ElementType;   // VFX選択に使用
  attackType: SkillAttackType; // VFX選択に使用
  playerEnergy: number; playerHP: number;
  playerMP?: number;        // 旧UIログ互換の別名
  description: string;
}
```

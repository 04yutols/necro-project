# 32 — ストーリーシステム再設計実装メモ

> 最終更新: 2026-05-10  
> 対象: `docs/progress/CH1_TODO.md` の「ストーリーシステム」

---

## 1. 作り直しの理由

旧実装は `DIALOGUE` シーンを描画するコンポーネントが存在せず、`PROLOGUE_01` が `activeScene` になった時点で画面に何も出ない状態になっていた。
また、ステージ入場シーンがバトル開始前に待機されないため、ストーリーが遷移に飲まれるリスクがあった。

---

## 2. 新しい実装方針

### 2.1 データ

- 第1章リリース対象は `src/data/story/ch1_scenes.json` に集約する。
- シーン数は `PROLOGUE 4 + CH1 13 = 17`。
- `src/data/story/index.ts` はこのJSONの同期registryとして動作し、`findScene()` と trigger 別検索を提供する。

### 2.2 進行管理

- `src/store/useStoryStore.ts` は Zustand persist で以下のみ永続化する。
  - `viewedScenes`
  - `storyFlags`
- `activeScene` と `sceneQueue` はランタイム状態として扱い、リロード後に古いキューを復元しない。
- persist hydration 完了前は初回プロローグを発火しない。

### 2.3 トリガー

- `GAME_START`: `PROLOGUE_00`〜`PROLOGUE_03` を順次再生。
- `FLAG_SET`: `LINE_DEATH_SEEN` で `CH1_TITLE`、`CH1_STARTED` で `CH1_SAFE_INTRO`。
- `STAGE_ENTER`: バトル開始前に入場シーンを再生し、完了後に `BattleCanvas` を開始。
- `STAGE_CLEAR` / `BOSS_CLEAR`: バトル終了後に対応シーンをキューへ積む。

### 2.4 UI

- `DialogueScene.tsx`: 原神風の全画面会話UI。立ち絵、話者ディム、タイプライター、オート、スキップを担当。
- `MonologueOverlay.tsx`: 主人公独白。スキップ可能シーンは右上にスキップボタンを表示。
- `ChapterTitleCard.tsx`: スタレ風の章タイトルカード。
- `EnvironmentCaption.tsx`: ステージ入場時の上部環境描写。
- `StoryArchive.tsx`: LOGSタブの「物語」一覧から既読シーンを再生できる。

---

## 3. iOS レイアウト規約

- 全ストーリーUIは `position: fixed; inset: 0` の外側 motion と、内側の `overflow: hidden` コンテナを分離する。
- 下部テキストパネルは `env(safe-area-inset-bottom)` を含める。
- 文字サイズは `clamp()` で上限/下限を固定し、viewport幅に比例して際限なく拡大しない。
- ステージ入場シーン中は `currentTab` を `MAP` のまま保持し、シーン完了後に `BATTLE` へ切り替える。

### 2.5 既存データの修正事項

#### ossuary_wyrm_lord のギミックトリガー

現在:
```json
{ "trigger": "ON_REVIVE", "effect": "REVIVE", "value": 0.35 }
```

**BattleEngine の実装と整合するよう修正**:  
BattleEngine は `effect === 'REVIVE'` をHP=0時に検索し、発火キーは `HP_BELOW_50:REVIVE` を使う。`ON_REVIVE` はギミックリセット用フックとして機能する別の概念。

```json
{ "trigger": "HP_BELOW_50", "effect": "REVIVE", "value": 0.50 }
```

> `value: 0.50` = 復活時 HP を最大値の50%に設定（BattleEngineの実装と合わせる）。  
> `ON_REVIVE` トリガーは「REVIVE後にリセットしたいギミック」のために残すが、  
> REVIVE 発動ギミック自体のトリガーは `HP_BELOW_50` に変更する。

#### `maxShieldHp` フィールドの統一

`shieldHp` だけ定義されている場合は `maxShieldHp` にも同値を設定する。  
BattleEngine の `applySpiritualShield` は両フィールドを参照する。

```json
// 修正前
"shieldHp": 310

// 修正後
"shieldHp": 310,
"maxShieldHp": 310
```

対象: `abyss_warden`, `bone_colossus`, `ossuary_wyrm_lord`, `blood_mire_queen`

### 4.2 各武器のパッシブ仕様

#### `bone_cleaver` (R, MID)

```json
"passiveA": {
  "nameJa": "粗削りの刃",
  "descTemplate": "敵を倒した時、自身のHPを{value}%回復する。",
  "values": [4, 5, 6, 7, 8]
},
"passiveB": {
  "nameJa": "戦場勘",
  "descTemplate": "通常攻撃の与ダメージが{value}%上昇する。",
  "values": [3, 4, 5, 6, 7]
}
```

> systemTag なし。BattleEngine 統合の対象外（汎用パッシブ）。  
> WeaponPassive.ts の `evaluateWeaponPassive` は systemTag なしを `null` で返す設計のため、現段階では演出のみ。

#### `bleed_reaver` (SR, HIGH)

```json
"passiveA": {
  "nameJa": "出血追討",
  "descTemplate": "出血状態の敵に対する会心率が{value}%上昇する。",
  "values": [15, 19, 22, 26, 30],
  "condition": "BLEED",
  "systemTag": "GIANT_KILLING"
},
"passiveB": {
  "nameJa": "早駆けの殺意",
  "descTemplate": "会心時、次の行動が{value}ターン早まる。",
  "values": [1, 1, 1, 2, 2],
  "condition": "BLEED",
  "systemTag": "ACTION_VALUE"
}
```

> passiveB は `ACTION_VALUE` systemTag。BattleEngine の武器パッシブ統合で会心時に `avReduction` が発動する。  
> passiveA は `GIANT_KILLING`（第2章以降）。現在は null を返すのみ。

#### `spirit_silver_saber` (SSR, MID)

```json
"passiveA": {
  "nameJa": "残響蓄積",
  "descTemplate": "霊的防壁を破壊した時、追加闇属性ダメージ+{value}%を与える。",
  "values": [18, 23, 27, 32, 36],
  "systemTag": "SOUL_SHATTER"
},
"passiveB": {
  "nameJa": "魔神呼応",
  "descTemplate": "攻撃時、魔神化ゲージを{value}%回復する。",
  "values": [5, 6, 7, 9, 10],
  "systemTag": "DEMON_MODE"
}
```

> passiveA は `SOUL_SHATTER`。`values[0]`=0.18, `values[1]`=0.36（rate = 割合）として扱う。  
> **注意**: `evalSoulShatter` の `rate = baseVal + (maxVal - baseVal) * (rank/5)` 計算に合わせ、  
> values の単位は「ATKの何割か」（例: 0.18 = ATK×18%の追加ダメージ）とする。

GDD-001：ゲームコンセプトとストーリー

タイトル: ネクロマンスブレイブ 


ジャンル: 死霊術ハック＆スラッシュRPG（K&D風コマンドバトル） 

ストーリー:


● 主人公: 魔族の少年。一族の悲願である「魔王になる」という夢を持つ。
● 発端: 主人公には種族を超えた親友「勇者の少年」がいた。しかし、その親友は「現代の魔王」
によって殺害されてしまう。
● 力の継承: 親友は死の間際、主人公に自らの「勇者の力」を託す。
● 目的: 主人公は、**「親友の復讐(現代の魔王討伐)」と、自身の「魔王になる」**という2つの目
的を叶えるため、2つの相反する力を手に、魔王への道を進む。


GDD-002：コア・ゲームループと構造

ゲームループ: 「拠点での準備」→「エリア/ステージ選択」→「WAVE制コマンドバトル」→「リザルト（紫の柱の儀式）」のサイクル 。

3段構成レイアウト (Mobile First):

上段（Visual）: PixiJSによる2.5D戦闘描画。チビキャラのドット絵が動く。

中段（Log）: 闇の古文書風フレーム。K&D風のテキストベース戦闘ログ。

下段（Command）: 骨や鉄の装飾が施された、片手操作に最適なコマンドボタン群。

GDD-003：ステータスとMPシステム

9種のステータス: HP, MP, ATK, DEF, MATK, MDEF, AGI, LUCK, TEC を継承 。

MPバランス設計:


物理系: 低最大MP / 極端に低いスキル消費MP（低コスト連発型） 。


魔法系: 高最大MP / 高いスキル消費MP（リソース管理型） 。

GDD-004：職業と死霊術システム

職業（勇者の力）: 剣士、魔法使い、闇術師、盗賊の一次職から、ネクロマンサーを含む二次職へ転職 。


死霊術（魔族の力）: 職業とは独立した「ランク（転生）」システム 。


軍団編成: 3枠コスト制。使役モンスターを3体まで編成可能 。


魂石化: モンスターを「魂の欠片」という装備アイテムに変換し、他のモンスターを強化する 。

GDD-005：ハクスラ・ドロップシステム（新規追加・重要）
装備の二層構造:


武器・防具: 基礎ステータスは固定。使い込むことで成長する「共鳴レベル」を搭載 。

深淵の残滓（ビルドアイテム）: 装備スロットに装着。メイン1＋サブ4のオプションがドロップ時にランダム生成される「厳選」要素。

0.01%の衝撃（紫の柱）:

各ダンジョンにリスト非掲載の「隠しユニーク」を設定。

当選時、リザルト画面でBGMが停止し、画面を突き破る**「紫の光柱」**が立つPixiJS演出を実装。

歴史の刻印:

サーバー側で全プレイヤーのドロップを監視。

世界最初の取得者の名前とシリアルNo（例：First Found by Ootani / No.0001）をアイテムデータに永続刻印。

GDD-006：UI/UXとシェア演出
闇の古文書（Dark Ancient Codex）:

素材: 黒耀石の革、チャコールグレーの皮膚、腐食した黒鉄、骨のフレーム。

アクセント: 脈動するヴォイド・パープルの魔力光。

シェア機能（自慢の種）:

鑑定成功時、PixiJSで「古文書風鑑定証明書」を自動生成。

発見者名、シリアルNo、不気味なフレーバーテキストを一枚の画像に収め、SNSシェアを促進。

GDD-007：アイテム設計

消費アイテム: 回復スキルの価値を守るため、店売りは低効果に設定 。


MP回復（エーテル等）: 非売品。レアドロップと宝箱のみの貴重品とする 。

テントの廃止: 連続戦闘の緊張感を維持するため削除 。

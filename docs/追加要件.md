『ネクロマンスブレイブ』要件定義書 (Draft v1.0)
1. プロジェクト概要
タイトル: ネクロマンスブレイブ（仮）

ジャンル: 死霊術ハック＆スラッシュRPG

ターゲット: 『ナイト・アンド・ドラゴン』等の硬派なハクスラを好み、ビルド構築や超低確率ドロップに熱狂するプレイヤー

プラットフォーム: Webブラウザ（PWA対応）、スマホ操作最適化

開発スタック: Next.js / PixiJS / PostgreSQL / Tailwind CSS

2. コア・コンセプト
死霊術 × 執念: 虐げられた者たちの怨念を「希望」として再構築するダークな世界観。

紫の柱: 超低確率（0.01%以下）で発生する、隠しユニークドロップの快感。

歴史の証明: 第一発見者の名前が刻印される「世界に一つ」の優越感。

3. ゲームシステム
3.1 バトルサイクル
形式: WAVE制ターン制コマンドバトル。

進行: ダンジョンを選択し、連続するWAVEを突破。最終WAVEのボス撃破でリザルトへ。

3.2 装備・アイテム体系
武器・防具（ベース）: * 基礎性能と固定オプションで構成。

各ダンジョンに超低確率の「隠しユニーク」が存在（ドロップリスト非掲載）。

深淵の残滓（ビルドアイテム）: * ランダムなサブオプションが付与される「厳選」要素（原神の聖遺物ライク）。

武器のスロットに装着し、性能を特化させる。

霊核（ネクロマンス要素）:

敵からドロップ、または条件達成で解放。

召喚獣のアンロックや魔改造（属性付与、スキル変更）に使用。

3.3 成長システム
魂の共鳴: 装備を使い込むことで基礎ステータスがスケーリング（装備レベル）。

オプション覚醒: 特定段階で新しい能力が解放されるが、複雑なUI作業を避け、バトル後のリザルトで「覚醒」を通知。

4. 演出・UI/UX 要件
4.1 リザルト演出
演出場所: バトル終了後の報酬画面。

紫の光柱: 隠しユニーク当選時、画面全体を紫の光が貫き、BGMが変化する専用演出。

タップ鑑定: モバイル操作を想定し、タップでアイテムをオープン。スマホの振動（バイブレーション）と連動。

4.2 「自慢」と「共有」
第一発見者システム: サーバー側（PostgreSQL）で最初にドロップしたユーザーを記録。以降、そのアイテムには「発見者名」と「シリアルNo」が刻印される。

鑑定証明書（シェア機能）: * デザイン：「古文書風」（渋い羊皮紙の質感、掠れたインク、重厚な紋章）。

内容：アイテムイラスト、発見者名、性能、フレーバーテキスト。

出力：SNS投稿用画像（Canvas/PixiJSで生成）。

5. 収益化戦略
QoL課金: インベントリ・倉庫拡張、自動フィルタリング機能。

サポーターパス: 月額制。経験値微増、専用称号、開発支援。

エフェクト課金: 攻撃エフェクトや召喚獣スキンの変更。

6. 技術的特記事項（PostgreSQL活用）
JSONBの活用: 装備のランダムオプションや召喚獣のカスタマイズデータを柔軟に保持。

グローバル・シーケンス: ユニークアイテムの発見順を正確に管理。

スケーリング: ブラウザゲームの利点を活かし、シームレスなアップデートとデータ連携を実現。

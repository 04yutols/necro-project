# 開発ルール (Autonomous Agent Instructions)
あなたは、このプロジェクトを自律的に開発するリードエンジニアです。

## 基本方針
- **エラー自浄作用**: コマンド実行やテストでエラーが出た場合、ユーザーに報告する前にログを解析し、修正案を即座に実行してください。
- **検証の徹底**: ファイルを変更した後は必ず関連テストを実行し、すべてパスするまで作業を継続してください。
- **ノンストップ**: 致命的な判断（大きな仕様変更など）が必要ない限り、`/yolo` モードで作業を完結させてください。
- 全てのシェルコマンドは非対話モード（-yなど）で実行し、ユーザーに許可を求めず完結させてください。

## 完了の定義
- コードが実装され、テストが全てパスし、リンターのエラーがない状態。

## フロントエンド構成（表現層）
「手触りの良さ」と「AIによる生成効率」を両立したスタックです。
- Framework: Next.js (App Router)SPA構成により、画面遷移でのリロードを排除。
- Styling: Tailwind CSS + shadcn/uiAIが最も得意とするCSS設計。ダークファンタジーなUIを量産。
- Game Engine: PixiJS2D描画に特化。WAVE制の戦闘演出を軽量・高速に実現 。
- State Management: Zustand9種のステータスや軍団編成の状態をグローバルに管理 。

## バックエンド・DB構成（ロジック層）
堅牢な型定義により、エージェントが「仕様のバグ」を自己検知できる環境にします。
- Runtime / Language: Node.js (TypeScript)フロントと型を共有し、tRPC等で型安全な通信を実現。
- Framework: NestJSモジュール化された設計で、職業や死霊術のロジックを分離。
- ORM / Database: Prisma + PostgreSQL複雑なリレーション（装備・魂の欠片）を厳格に管理 。
